Session Manager

Secure shell access without SSH

Session Manager Architecture

Session Manager provides secure, auditable node management without the need to open inbound ports, maintain bastion hosts, or manage SSH keys.

Key Security Features:

  • No SSH Keys: Eliminates key management overhead
  • No Bastion Hosts: Reduces attack surface
  • No Inbound Ports: Enhanced network security
  • IAM Integration: Centralized access control
  • Audit Logging: Complete session recording

Connection Process:

  1. User initiates session via console/CLI
  2. IAM validates permissions
  3. SSM Agent establishes secure tunnel
  4. Session data encrypted in transit
  5. All activity logged to CloudWatch/S3

Supported Platforms:

  • Amazon Linux, Ubuntu, RHEL, SUSE
  • Windows Server 2008 R2 and later
  • macOS (with SSM Agent)
  • On-premises servers

Advanced Configuration

Session Preferences:

Shell Preferences

  • Default shell selection (/bin/bash, /bin/sh)
  • Environment variables
  • Working directory
  • Session timeout settings

Logging Configuration

  • CloudWatch Logs integration
  • S3 bucket for session logs
  • Encryption with KMS keys
  • Log retention policies

Access Control:

  • Instance-level: Tag-based access control
  • User-level: IAM policy conditions
  • Time-based: Temporary access grants
  • Command filtering: Restrict specific commands

Port Forwarding:

  • Local port forwarding to remote services
  • Database connection tunneling
  • Application debugging access
  • Secure remote desktop connections

Implementation & Best Practices

IAM Policy Example

"Effect": "Allow",
"Action": [
"ssm:StartSession"
],
"Resource": [
"arn:aws:ec2:*:*:instance/*"
],
"Condition": {
"StringEquals": {
"aws:RequestedRegion": "us-east-1"
}
}

Security Best Practices

  • Enable session logging
  • Use least privilege IAM policies
  • Implement session timeouts
  • Monitor session activity
  • Use VPC endpoints for private access
  • Enable MFA for sensitive operations

Troubleshooting

  • Verify SSM Agent status
  • Check IAM permissions
  • Validate VPC endpoints
  • Review CloudWatch logs
  • Test network connectivity
  • Confirm instance registration
