Subnets

Public & Private Subnets

What are Subnets?

Subnets are subdivisions of a VPC that reside in a specific Availability Zone. They allow you to segment your network for better organization and security.

  • Each subnet must reside in a single AZ
  • Subnets cannot span multiple AZs
  • Multiple subnets can exist in the same AZ
  • CIDR block must be subset of VPC CIDR
  • AWS reserves 5 IP addresses per subnet

Public Subnets

Public subnets have a route to an Internet Gateway, allowing instances to communicate with the internet.

Characteristics

  • Route table points to Internet Gateway
  • Instances can have public IP addresses
  • Direct internet connectivity
  • Used for web servers, load balancers

Private Subnets

Private subnets do not have direct internet access. They use NAT Gateway/Instance for outbound internet connectivity.

Characteristics

  • No direct internet access
  • Route through NAT for outbound traffic
  • Enhanced security
  • Used for databases, application servers

Reserved IP Addresses

AWS reserves 5 IP addresses in each subnet that cannot be assigned to instances.

Example: 10.0.0.0/24

  • 10.0.0.0 - Network address
  • 10.0.0.1 - VPC router
  • 10.0.0.2 - DNS server
  • 10.0.0.3 - Future use
  • 10.0.0.255 - Broadcast address

Exam Strategy Tip

Remember: Subnets are AZ-specific. Public subnets have IGW route. Private subnets use NAT for outbound. AWS reserves 5 IPs per subnet. Plan CIDR blocks to avoid overlap.
