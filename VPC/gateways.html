Gateways

Internet Gateway & NAT

Internet Gateway (IGW)

Provides internet access to public subnets. Horizontally scaled, redundant, and highly available.

  • One IGW per VPC
  • Enables bidirectional internet access
  • Performs NAT for public IP addresses
  • No bandwidth constraints
  • No availability risk or bandwidth constraints

NAT Instance

EC2 instance that provides NAT functionality for private subnets. You manage the instance.

Characteristics

  • Must be in public subnet
  • Disable source/destination check
  • Must manage security groups
  • Can be used as bastion host
  • Supports port forwarding

NAT Gateway

Fully managed NAT service that provides outbound internet access for private subnets.

Benefits

  • Fully managed by AWS
  • Higher bandwidth (up to 45 Gbps)
  • Better availability
  • No security groups needed
  • Automatic software patching

NAT Gateway vs NAT Instance

NAT Gateway

  • Managed service
  • Higher availability
  • Higher bandwidth
  • No maintenance required
  • Higher cost

NAT Instance

  • Self-managed EC2 instance
  • Lower cost
  • More customizable
  • Can use as bastion host
  • Single point of failure

Exam Strategy Tip

Remember: IGW for public subnet internet access. NAT Gateway is managed and preferred for production. NAT Instance is cheaper but requires management. One IGW per VPC limit.
