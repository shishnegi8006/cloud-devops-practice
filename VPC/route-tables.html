Route Tables

Traffic Routing Control

What are Route Tables?

Route tables contain rules (routes) that determine where network traffic is directed. Each subnet must be associated with a route table.

  • Control traffic routing within VPC
  • Each subnet associated with one route table
  • Route table can be associated with multiple subnets
  • Most specific route takes precedence
  • Local route cannot be deleted

Main Route Table

Automatically created with VPC. Default route table for subnets without explicit association.

Default Routes

  • Local route for VPC CIDR
  • Cannot be deleted
  • Used by subnets without explicit association
  • Can be modified to add custom routes

Custom Route Tables

User-created route tables for specific routing requirements. Best practice for production environments.

Public Route Table

0.0.0.0/0 → Internet Gateway

Private Route Table

0.0.0.0/0 → NAT Gateway/Instance

Route Priority

Routes are evaluated based on longest prefix match (most specific route wins).

Priority Order

  • Local routes (highest priority)
  • Most specific routes (/32, /24, /16)
  • Static routes
  • Propagated routes
  • Default route (0.0.0.0/0)

Exam Strategy Tip

Remember: Most specific route wins. Local routes have highest priority. Each subnet needs one route table. Main route table is default. Use custom route tables for production.
