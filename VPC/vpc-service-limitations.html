VPC Service Limitations

Understanding VPC Quotas and Constraints

VPC Limits

  • VPCs per Region: 5 (default), up to 100 (soft limit)
  • Subnets per VPC: 200 (default)
  • IPv4 CIDR blocks per VPC: 5 (default)
  • IPv6 CIDR blocks per VPC: 1
  • Route tables per VPC: 200 (default)
  • Routes per route table: 50 (default), up to 1,000
  • Elastic IPs per Region: 5 (default)
  • Customer gateways per Region: 50

Security Limits

  • Security groups per VPC: 2,500 (default)
  • Rules per security group: 60 (default)
  • Security groups per network interface: 5
  • Network ACLs per VPC: 200 (default)
  • Rules per network ACL: 20 (default)
  • Network interfaces per instance: Varies by instance type
  • Private IP addresses per network interface: Varies by instance type

Gateway Limits

  • Internet gateways per Region: 5 (default)
  • NAT gateways per Availability Zone: 5 (default)
  • VPN connections per Region: 50
  • VPN connections per VPC: 10
  • Virtual private gateways per Region: 5
  • Customer gateways per Region: 50
  • Transit gateways per account per Region: 5
  • Transit gateway attachments per transit gateway: 5,000

Connectivity Limits

  • VPC peering connections per VPC: 50 (default), up to 125
  • Outstanding VPC peering connection requests: 25
  • Expiry time for unaccepted VPC peering connection request: 1 week
  • VPC endpoints per VPC: 255
  • Interface VPC endpoints per VPC: 50
  • Gateway VPC endpoints per VPC: 255
  • Policy size for VPC endpoint: 20,480 characters

Network Performance

  • NAT Gateway bandwidth: Up to 45 Gbps
  • VPN connection throughput: Up to 1.25 Gbps
  • Direct Connect bandwidth: 50 Mbps to 100 Gbps
  • Enhanced networking (SR-IOV): Up to 100 Gbps
  • Placement groups: Up to 25 Gbps between instances
  • Inter-AZ bandwidth: Up to 25 Gbps
  • Intra-AZ bandwidth: Up to 100 Gbps

DNS and DHCP Limits

  • DHCP option sets per Region: 100
  • DNS resolution: Enabled by default
  • DNS hostnames: Disabled by default
  • DNS queries per second per network interface: 1,024
  • Maximum DNS response size: 4,096 bytes
  • Private hosted zones per VPC: 100
  • Rules per resolver rule group: 1,000

Exam Tips for VPC Limitations

  • Most VPC limits are soft limits that can be increased via AWS Support
  • Plan for security group rules carefully - 60 rules per group is the default limit
  • VPC peering has a limit of 50 connections per VPC - consider Transit Gateway for complex topologies
  • NAT Gateway bandwidth scales automatically up to 45 Gbps
  • Route table limits affect complex routing scenarios - plan accordingly
  • Remember that some limits are per Region, others per VPC or per resource
