VPC Connectivity

Peering & Transit Gateway

VPC Peering

Network connection between two VPCs that enables routing traffic using private IP addresses.

  • One-to-one connection between VPCs
  • Can peer across regions and accounts
  • No transitive peering
  • CIDR blocks cannot overlap
  • Must update route tables and security groups

Transit Gateway

Central hub for connecting VPCs, on-premises networks, and other AWS services.

Benefits

  • Simplifies network architecture
  • Supports transitive routing
  • Centralized connectivity hub
  • Scales to thousands of connections

VPC Endpoints

Private connection to AWS services without internet gateway or NAT device.

Gateway Endpoints

S3 and DynamoDB only, free

Interface Endpoints

Most AWS services, powered by PrivateLink

Peering vs Transit Gateway

VPC Peering

  • One-to-one connections
  • No transitive routing
  • Lower cost for simple setups
  • Complex with many VPCs

Transit Gateway

  • Hub-and-spoke model
  • Transitive routing supported
  • Scales better
  • Higher cost but simpler management

Exam Strategy Tip

Remember: VPC Peering is one-to-one, no transitive routing. Transit Gateway is hub-and-spoke with transitive routing. VPC Endpoints provide private AWS service access. CIDR blocks cannot overlap in peering.
