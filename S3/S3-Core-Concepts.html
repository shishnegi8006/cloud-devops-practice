S3 Core Concepts

Fundamental concepts of Amazon Simple Storage Service

S3 Overview

What is Amazon S3?

Object Storage: Store and retrieve any amount of data from anywhere

Scalability: Virtually unlimited storage capacity

Durability: 99.999999999% (11 9's) durability

Key Characteristics

Global Service: Buckets are region-specific but namespace is global

Web Interface: RESTful API and web-based management

Pay-as-you-go: Only pay for storage and requests used

Buckets and Objects

S3 Buckets

Containers for objects with globally unique names.

Naming Rules:

  • • 3-63 characters long
  • • Lowercase letters, numbers, hyphens
  • • No uppercase or underscores
  • • Must start/end with letter or number

S3 Objects

Files stored in S3 buckets with metadata and unique keys.

Components:

  • • Key (object name/path)
  • • Value (object data)
  • • Version ID (if versioning enabled)
  • • Metadata (key-value pairs)

Object Keys

Unique identifiers for objects within a bucket.

Examples:

  • • my-file.jpg
  • • folder/subfolder/file.txt
  • • logs/2023/12/app.log

S3 Versioning

Version Control

Keep multiple versions of objects in the same bucket.

States:

  • • Unversioned (default)
  • • Versioning-enabled
  • • Versioning-suspended

Protection Benefits

Protects against accidental deletion and modification.

Protection:

  • • Accidental overwrites
  • • Accidental deletions
  • • Application errors

Delete Behavior

Special handling of delete operations with versioning.

Delete Types:

  • • Simple delete: adds delete marker
  • • Permanent delete: specify version ID
