S3 Storage Classes

S3 Standard

High durability and availability for frequently accessed data.

Use Case:

Website content, mobile apps, data analytics

Availability: 99.99% | Durability: 99.999999999%

S3 Standard-IA

Lower cost for infrequently accessed data with rapid access.

Use Case:

Backups, disaster recovery, long-term storage

Cost: 50% less than Standard | Min Duration: 30 days

S3 Glacier

Low-cost archival storage with retrieval times from minutes to hours.

Use Case:

Long-term archives, compliance, digital preservation

Retrieval: 1-5 minutes to 12 hours | Min Duration: 90 days

S3 Glacier Deep Archive

Lowest cost storage for rarely accessed data.

Use Case:

Regulatory compliance, media archives, backup

Retrieval: 12-48 hours | Min Duration: 180 days

S3 Intelligent-Tiering

Automatically moves data between access tiers based on usage patterns.

Use Case:

Unknown or changing access patterns

Monitoring Fee: $0.0025 per 1,000 objects

Core S3 Features

Versioning

Keep multiple versions of objects for data protection and recovery.

Use Case:

Protect against accidental deletion or modification

Benefit: Easy rollback to previous versions

Lifecycle Management

Automatically transition or delete objects based on age.

Use Case:

Move old logs to cheaper storage after 30 days

Benefit: Automatic cost optimization

Cross-Region Replication

Automatically replicate objects to another AWS region.

Use Case:

Disaster recovery, compliance, reduced latency

Requirement: Versioning must be enabled

Encryption

Server-side and client-side encryption options for data protection.

Options:

SSE-S3, SSE-KMS, SSE-C, Client-side encryption

Default: SSE-S3 encryption enabled by default

Multipart Upload

Upload large files in parts for improved performance and reliability.

Use Case:

Upload large video files, database backups

Recommended: Files larger than 100MB

Transfer Acceleration

Use CloudFront edge locations to speed up uploads.

Use Case:

Global users uploading large files

Benefit: Up to 50-500% faster uploads

S3 Access Control

Bucket Policies

JSON-based policies attached to buckets.

Use Case:

Grant public read access to website assets

Access Control Lists

Legacy method for granting basic permissions.

Use Case:

Simple read/write permissions

IAM Policies

User and role-based access control.

Use Case:

Grant specific users access to buckets

Pre-signed URLs

Temporary URLs for secure access.

Use Case:

Allow users to upload files directly
