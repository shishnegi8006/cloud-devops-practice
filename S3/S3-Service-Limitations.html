Bucket Naming Constraints

Bucket Name Requirements

  • • Must be 3-63 characters long
  • • Must be globally unique across all AWS accounts
  • • Must start and end with lowercase letter or number
  • • Cannot contain uppercase letters or underscores
  • • Cannot be formatted as IP address (e.g., 192.168.1.1)

Object Key Constraints

Object keys can be up to 1,024 characters long. Safe characters include letters, numbers, and special characters like !, -, _, ., *, ', (, and ).

Multipart Upload Constraints

  • • Maximum 10,000 parts per multipart upload
  • • Part numbers: 1 to 10,000
  • • Mandatory for objects larger than 5 GB
  • • Recommended for objects larger than 100 MB

Regional & Transfer Constraints

Bucket Region Binding

Buckets are created in a specific region and cannot be moved to another region. Objects must be copied to change regions.

Transfer Acceleration

Not available in all regions. Cannot be used with buckets that have periods (.) in the name.

VPC Endpoint Limitations

Gateway VPC endpoints for S3 are region-specific and only work within the same region as the bucket.

Important Considerations

Eventually Consistent: S3 provides strong read-after-write consistency for all operations as of December 2020.

Soft Limits: Many S3 limits can be increased by contacting AWS Support, but some are architectural constraints.

Storage Classes: Not all storage classes support all features. Check compatibility before implementation.
