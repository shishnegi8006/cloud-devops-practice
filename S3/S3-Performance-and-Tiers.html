S3 Performance & Tiers

Storage classes, lifecycle management, and performance optimization

S3 Storage Classes

S3 Standard

General-purpose storage for frequently accessed data.

Characteristics:

  • • 99.99% availability SLA
  • • 99.999999999% durability
  • • Low latency, high throughput
  • • No retrieval fees
Best For: Frequently accessed data, cloud applications

S3 Intelligent-Tiering

Automatically moves data between access tiers based on usage patterns.

Tiers:

  • • Frequent Access (default)
  • • Infrequent Access (30 days)
  • • Archive Instant Access (90 days)
  • • Archive Access (90-270 days)
  • • Deep Archive Access (180-730 days)
Best For: Unknown or changing access patterns

S3 Standard-IA

For infrequently accessed data that requires rapid access when needed.

Characteristics:

  • • 99.9% availability SLA
  • • Lower storage cost than Standard
  • • Retrieval fees apply
  • • 30-day minimum storage duration
Best For: Backups, disaster recovery, long-term storage

S3 One Zone-IA

Lower-cost option for infrequently accessed data that doesn't require multiple AZ resilience.

Characteristics:

  • • 99.5% availability SLA
  • • Single AZ storage
  • • 20% less cost than Standard-IA
  • • 30-day minimum storage duration
Best For: Secondary backup copies, recreatable data

S3 Glacier Instant Retrieval

Archive storage with millisecond retrieval for rarely accessed data.

Characteristics:

  • • Millisecond retrieval
  • • 99.9% availability SLA
  • • 90-day minimum storage duration
  • • 68% lower cost than Standard-IA
Best For: Archive data needing immediate access

S3 Glacier Flexible Retrieval

Low-cost archive storage for data accessed 1-2 times per year.

Retrieval Options:

  • • Expedited: 1-5 minutes
  • • Standard: 3-5 hours
  • • Bulk: 5-12 hours
  • • 90-day minimum storage duration
Best For: Backup, archive, disaster recovery

S3 Glacier Deep Archive

Lowest-cost storage class for long-term retention and digital preservation.

Characteristics:

  • • Standard: 12 hours retrieval
  • • Bulk: 48 hours retrieval
  • • 180-day minimum storage duration
  • • Lowest cost storage class
Best For: Compliance archives, digital preservation

S3 Outposts

Object storage on-premises for local data residency requirements.

Features:

  • • On-premises S3 storage
  • • Local data processing
  • • S3 APIs and features
  • • Data residency compliance
Best For: Local data residency, low latency access

S3 Express One Zone

High-performance storage for latency-sensitive applications.

Performance:

  • • Single-digit millisecond latency
  • • Up to 10x faster performance
  • • Single AZ deployment
  • • Request cost optimization
Best For: High-performance computing, analytics

Lifecycle Management

Transition Rules

Automatically move objects between storage classes based on age.

Common Transitions:

  • • Standard → Standard-IA (30 days)
  • • Standard-IA → Glacier (90 days)
  • • Glacier → Deep Archive (180 days)

Expiration Rules

Automatically delete objects after specified time periods.

Options:

  • • Delete current versions
  • • Delete previous versions
  • • Delete incomplete multipart uploads
  • • Delete expired delete markers

Rule Filters

Apply lifecycle rules to specific objects using filters.

Filter Types:

  • • Prefix filters (folder paths)
  • • Object tags
  • • Object size
  • • Combination filters

Performance Optimization

Multipart Upload

Upload large objects in parts for improved performance and reliability.

Benefits:

  • • Parallel uploads
  • • Resume failed uploads
  • • Upload while creating object
  • • Required for objects >5GB
Recommended: Use for objects >100MB

Transfer Acceleration

Use CloudFront edge locations to accelerate uploads to S3.

How it works:

  • • Upload to nearest edge location
  • • Transfer over AWS backbone
  • • Up to 50-500% faster
  • • Additional cost per GB
Best For: Global users, large file uploads

S3 Select

Retrieve only a subset of data from objects using SQL expressions.

Supported Formats:

  • • CSV, JSON, Parquet
  • • GZIP, BZIP2 compression
  • • Server-side filtering
  • • Up to 400% faster
Benefit: Reduce data transfer and costs

Byte-Range Fetches

Download specific byte ranges of objects for parallel processing.

Benefits:

  • • Parallel downloads
  • • Resume failed downloads
  • • Partial file retrieval
  • • Improved resilience
Use Case: Large file downloads, media streaming

Request Patterns

Optimize object key naming for better performance.

Best Practices:

  • • Avoid sequential patterns
  • • Use random prefixes
  • • Distribute request load
  • • Consider hexadecimal prefixes
Goal: Achieve 3,500 PUT/COPY/POST/DELETE and 5,500 GET/HEAD per prefix per second

CloudFront Integration

Use CloudFront CDN for improved global access performance.

Benefits:

  • • Global edge locations
  • • Caching at edge
  • • Reduced latency
  • • Lower S3 request costs
Best For: Static websites, content distribution
