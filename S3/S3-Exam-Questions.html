Storage Classes & Cost Optimization

A company stores 100TB of data in S3 Standard. The data is accessed frequently for the first 30 days, then rarely accessed but must be available within minutes when needed. After 90 days, the data is archived and accessed only once per year. What is the most cost-effective storage strategy?

A) Keep all data in S3 Standard for immediate access
B) Use S3 Intelligent-Tiering with lifecycle policy: Standard → IA (30 days) → Glacier (90 days)
C) Move all data to S3 Glacier immediately
D) Use S3 One Zone-IA for all data

Explanation:

Correct Answer: B
S3 Intelligent-Tiering automatically moves data between access tiers based on usage patterns, eliminating the guesswork. The lifecycle policy ensures data transitions to cheaper storage classes: Standard (0-30 days for frequent access) → Standard-IA (30-90 days for infrequent access with minute retrieval) → Glacier (90+ days for archival with hour retrieval). This provides the most cost-effective solution while meeting access requirements.

Security & Access Control

A web application needs to allow users to upload files directly to S3 without exposing AWS credentials. The uploads should be limited to specific file types and sizes. What is the best approach?

A) Create IAM users for each application user
B) Use pre-signed URLs with conditions for file type and size
C) Make the S3 bucket public and use client-side validation
D) Use S3 Transfer Acceleration with API Gateway

Explanation:

Correct Answer: B
Pre-signed URLs provide temporary, secure access to S3 objects without exposing AWS credentials. You can include conditions in the pre-signed URL policy to restrict file types (content-type), file sizes (content-length-range), and other parameters. This approach is secure, scalable, and doesn't require creating individual IAM users or making buckets public.

Performance & Scalability

A global application experiences slow S3 upload speeds from users in different regions. The application uploads large video files (1-5GB each). What combination of solutions would improve performance?

A) Use S3 Standard-IA storage class
B) Enable S3 Transfer Acceleration and use multipart upload
C) Create multiple S3 buckets in different regions
D) Use CloudFront for uploads

Explanation:

Correct Answer: B
S3 Transfer Acceleration uses CloudFront edge locations to speed up uploads from global users. Multipart upload is recommended for files larger than 100MB and allows parallel uploads of file parts, significantly improving performance for large files. This combination addresses both the global performance issue and large file upload optimization.

Data Protection & Versioning

A company needs to protect critical data in S3 from accidental deletion and ensure compliance with data retention policies. They need to keep data for 7 years but want to optimize costs. What is the best approach?

A) Enable versioning only
B) Enable versioning, MFA Delete, and lifecycle policy with Glacier Deep Archive
C) Use S3 Object Lock only
D) Create daily backups to another S3 bucket

Explanation:

Correct Answer: B
This comprehensive approach provides: Versioning (protects against accidental overwrites), MFA Delete (prevents unauthorized deletion of versions), and lifecycle policy transitioning to Glacier Deep Archive (lowest cost for long-term retention). This combination ensures data protection, compliance, and cost optimization for the 7-year retention requirement.

5

Cross-Region Replication

A financial services company needs to replicate critical data from their primary S3 bucket in us-east-1 to eu-west-1 for disaster recovery. The data must be encrypted and the replication should be automatic. What configuration is required?

A) Enable Cross-Region Replication only
B) Enable versioning on both buckets, create IAM role, configure CRR with SSE-KMS
C) Use S3 Sync command with encryption
D) Set up Lambda function to copy objects

Explanation:

Correct Answer: B
Cross-Region Replication (CRR) requires: 1) Versioning enabled on both source and destination buckets, 2) IAM role with permissions to read from source and write to destination, 3) CRR configuration specifying the destination bucket. Adding SSE-KMS ensures encryption in transit and at rest. This provides automatic, secure replication for disaster recovery.
