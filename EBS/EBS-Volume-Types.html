EBS Volume Types

Complete guide to all EBS volume types and their characteristics

SSD-Based Volumes

gp3 (General Purpose SSD)

Performance

  • • Baseline: 3,000 IOPS
  • • Max: 16,000 IOPS
  • • Throughput: 125-1,000 MiB/s
  • • Size: 1 GiB - 16 TiB

Use Cases

  • • Boot volumes
  • • Low-latency interactive apps
  • • Development and test environments
  • • Virtual desktops
Key Feature: Independent IOPS and throughput provisioning

gp2 (General Purpose SSD)

Performance

  • • Baseline: 3 IOPS per GiB
  • • Min: 100 IOPS
  • • Max: 16,000 IOPS
  • • Burst: Up to 3,000 IOPS

Use Cases

  • • System boot volumes
  • • Small to medium databases
  • • Development environments
  • • Cost-sensitive workloads
Key Feature: Burstable performance with credit system

io2 (Provisioned IOPS SSD)

Performance

  • • IOPS: 100 - 64,000
  • • Ratio: 500 IOPS per GiB
  • • Throughput: Up to 1,000 MiB/s
  • • Size: 4 GiB - 16 TiB

Use Cases

  • • Critical business applications
  • • Large database workloads
  • • I/O-intensive applications
  • • Multi-Attach scenarios
Key Feature: 99.999% durability and Multi-Attach support

io1 (Provisioned IOPS SSD)

Performance

  • • IOPS: 100 - 64,000
  • • Ratio: 50 IOPS per GiB
  • • Throughput: Up to 1,000 MiB/s
  • • Size: 4 GiB - 16 TiB

Use Cases

  • • High-performance databases
  • • Mission-critical applications
  • • Sustained IOPS performance
  • • Legacy applications
Key Feature: Consistent, predictable performance

HDD-Based Volumes

st1 (Throughput Optimized HDD)

Performance

  • • Baseline: 40 MiB/s per TiB
  • • Burst: 250 MiB/s per TiB
  • • Max: 500 MiB/s
  • • Size: 125 GiB - 16 TiB

Use Cases

  • • Big data analytics
  • • Data warehouses
  • • Log processing
  • • Sequential workloads
Key Feature: Optimized for sequential, throughput-intensive workloads

sc1 (Cold HDD)

Performance

  • • Baseline: 12 MiB/s per TiB
  • • Burst: 80 MiB/s per TiB
  • • Max: 250 MiB/s
  • • Size: 125 GiB - 16 TiB

Use Cases

  • • Infrequently accessed data
  • • Archive storage
  • • Backup storage
  • • Cost-sensitive scenarios
Key Feature: Lowest cost per GB for infrequent access

Volume Selection Guidelines

Performance Requirements

High IOPS: Choose io2/io1 for >16,000 IOPS

High Throughput: Use st1 for sequential workloads

Balanced: gp3 for most general-purpose needs

Cost Optimization

Budget-conscious: gp2 or sc1 for cost savings

Archive: sc1 for infrequently accessed data

Performance/Cost: gp3 offers best balance
