EBS Multi-Attach

Share EBS volumes across multiple EC2 instances

Multi-Attach Overview

How Multi-Attach Works

Shared access: Single volume attached to multiple instances

Block-level: Raw block device shared at storage level

Concurrent: All instances can read/write simultaneously

Key Limitations

Volume types: Only io1 and io2 volumes supported

Instance limit: Maximum 16 instances per volume

Same AZ: All instances must be in same Availability Zone

Technical Requirements

Volume Requirements

Specific EBS volume types and configurations required for Multi-Attach.

Supported Types:

  • • io1 volumes (legacy)
  • • io2 volumes (recommended)
  • • Provisioned IOPS required

Not Supported:

  • • gp2, gp3 volumes
  • • st1, sc1 volumes
  • • Root volumes

Instance Requirements

EC2 instance specifications and configurations for Multi-Attach support.

Instance Types:

  • • Nitro System instances only
  • • R5, M5, C5, and newer generations
  • • EBS-optimized recommended

Networking:

  • • Enhanced networking support
  • • SR-IOV enabled

File System Requirements

Cluster-aware file systems required to prevent data corruption.

Supported Systems:

  • • Oracle Cluster File System (OCFS2)
  • • Red Hat Global File System (GFS2)
  • • Pacemaker with DRBD

Not Supported:

  • • Standard file systems (ext4, xfs)
  • • NTFS, FAT32

Implementation Considerations

Performance Considerations

IOPS sharing: Total IOPS shared across all attached instances

Network overhead: Cluster coordination adds latency

Load balancing: Distribute I/O across instances

Data Consistency

Locking: Distributed locking prevents conflicts

Synchronization: Cluster coordination required

Risk: Data corruption without proper file system

Configuration Steps

Volume Setup

Step 1: Create Volume

  • 1. Choose io1 or io2 volume type
  • 2. Provision required IOPS
  • 3. Enable Multi-Attach option
  • 4. Configure encryption if needed

Step 2: Attach to Instances

  • 1. Attach volume to first instance
  • 2. Attach same volume to additional instances
  • 3. Verify attachment on all instances
  • 4. Note device names may differ

File System Setup

Step 3: Install Cluster Software

  • 1. Install cluster-aware file system
  • 2. Configure cluster membership
  • 3. Set up distributed locking
  • 4. Test cluster communication

Step 4: Format and Mount

  • 1. Format volume with cluster file system
  • 2. Mount on all cluster nodes
  • 3. Configure automatic mounting
  • 4. Test concurrent access
