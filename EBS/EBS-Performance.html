EBS Performance

Optimize IOPS, throughput, and monitoring for EBS volumes

Performance Fundamentals

IOPS vs Throughput

IOPS: Input/Output Operations Per Second - measures random access performance

Throughput: Amount of data transferred per second - measures sequential access

Balance: Choose based on workload pattern - random vs sequential

Performance Factors

Volume Type: gp3, io2 offer highest performance

Volume Size: Larger volumes generally provide better performance

Instance Type: EBS-optimized instances provide dedicated bandwidth

IOPS Optimization

Provisioned IOPS

io1 and io2 volumes allow you to provision specific IOPS independent of volume size.

io2 Ratios:

Up to 500 IOPS per GiB (max 64,000 IOPS)

io1 Ratios:

Up to 50 IOPS per GiB (max 64,000 IOPS)

Best For: Consistent, predictable performance requirements

Burstable IOPS

gp2 volumes use a credit system to burst above baseline performance.

Baseline:

3 IOPS per GiB (minimum 100 IOPS)

Burst:

Up to 3,000 IOPS using I/O credits

Credit Rate: 3 credits per GiB per second

gp3 Flexibility

gp3 allows independent provisioning of IOPS and throughput.

Baseline:

3,000 IOPS and 125 MiB/s included

Maximum:

16,000 IOPS and 1,000 MiB/s

Advantage: Pay only for performance you need

Throughput Optimization

st1 Throughput

Throughput Optimized HDD designed for sequential workloads.

Baseline:

40 MiB/s per TiB

Burst:

250 MiB/s per TiB (max 500 MiB/s)

Best For: Big data, data warehouses, log processing

Large Block Sizes

Optimize I/O block sizes for better throughput performance.

Recommended:

1 MiB block sizes for sequential workloads

Impact:

Reduces overhead and improves efficiency

Tools: Use fio or dd for testing

RAID Configurations

Combine multiple EBS volumes for higher aggregate performance.

RAID 0:

Stripe for performance (no redundancy)

RAID 1:

Mirror for redundancy (no performance gain)

Note: Consider single large volume first

EBS-Optimized Instances

Dedicated Bandwidth

Isolation: Separate network bandwidth for EBS traffic

Consistency: Predictable performance without network contention

Availability: Default on newer instance types

Bandwidth Specifications

m5.large: Up to 4,750 Mbps

m5.xlarge: Up to 4,750 Mbps

m5.24xlarge: 19,000 Mbps

Performance Monitoring

CloudWatch Metrics

Monitor EBS performance using built-in CloudWatch metrics.

Key Metrics:

  • • VolumeReadOps/VolumeWriteOps
  • • VolumeReadBytes/VolumeWriteBytes
  • • VolumeThroughputPercentage
  • • VolumeConsumedReadWriteOps

Performance Alerts

Set up CloudWatch alarms for performance thresholds.

Alert Conditions:

  • • IOPS utilization > 80%
  • • Throughput utilization > 80%
  • • Queue depth consistently high
  • • I/O credit balance low (gp2)

Performance Testing

Use tools to benchmark and validate EBS performance.

Tools:

  • • fio - Flexible I/O tester
  • • iostat - I/O statistics
  • • iotop - I/O monitoring
  • • dd - Basic throughput testing

Performance Best Practices

Optimization Strategies

Pre-warm volumes: Initialize new volumes from snapshots

Queue depth: Maintain optimal queue depth (1-32)

I/O size: Use appropriate block sizes for workload

Common Pitfalls

Avoid: Small random I/O on st1/sc1 volumes

Avoid: Exceeding instance EBS bandwidth limits

Avoid: Ignoring I/O credit depletion on gp2
