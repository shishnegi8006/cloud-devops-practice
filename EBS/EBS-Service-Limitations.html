EBS Service Limitations

Elastic Block Store Quotas and Constraints

Volume Limits

  • EBS volumes per Region: 5,000 (default)
  • EBS volumes per instance: No limit (depends on instance type)
  • Total EBS storage per Region: 300 TiB (default)
  • EBS snapshots per Region: 100,000
  • Concurrent snapshot copies per destination Region: 20
  • EBS Multi-Attach volumes per Region: 5 (default)
  • Instances per Multi-Attach volume: 16
  • Fast Snapshot Restore per snapshot: 50 AZs

Volume Size Limits

  • gp3 volumes: 1 GiB - 16 TiB
  • gp2 volumes: 1 GiB - 16 TiB
  • io2 Block Express: 4 GiB - 64 TiB
  • io2 volumes: 4 GiB - 16 TiB
  • io1 volumes: 4 GiB - 16 TiB
  • st1 volumes: 125 GiB - 16 TiB
  • sc1 volumes: 125 GiB - 16 TiB
  • Root volume size: 1 GiB - 16 TiB (varies by AMI)

Performance Limits

  • gp3 max IOPS: 16,000 IOPS
  • gp3 max throughput: 1,000 MiB/s
  • gp2 max IOPS: 16,000 IOPS
  • io2 Block Express max IOPS: 256,000 IOPS
  • io2 max IOPS: 64,000 IOPS
  • io1 max IOPS: 64,000 IOPS
  • st1 max throughput: 500 MiB/s
  • sc1 max throughput: 250 MiB/s

Instance Attachment Limits

  • Nitro instances: 28 EBS volumes (including root)
  • Non-Nitro instances: 40 EBS volumes (including root)
  • EBS bandwidth per instance: Up to 80 Gbps (varies by type)
  • EBS-optimized bandwidth: Dedicated EBS bandwidth
  • Boot volume attachment: 1 per instance
  • Multi-Attach supported instances: Nitro-based only
  • Multi-Attach supported volume types: io1, io2 only
  • Multi-Attach same AZ requirement: Yes

Snapshot Limits

  • Snapshots per Region: 100,000
  • Concurrent snapshots per volume: 1
  • Snapshot copy requests per destination Region: 20
  • Fast Snapshot Restore per snapshot: 50 AZs
  • Snapshot sharing: Up to 1,000 AWS accounts
  • Public snapshot sharing: Supported
  • Cross-region snapshot copy: Supported
  • Incremental snapshot storage: Only changed blocks

Encryption & Security Limits

  • KMS keys per Region: 10,000 customer managed keys
  • Encryption by default: Per Region setting
  • Cross-region encrypted snapshot copy: Supported
  • Encryption key change: Not supported after creation
  • Unencrypted to encrypted conversion: Via snapshot copy
  • In-transit encryption: Between instance and EBS
  • Performance impact of encryption: Minimal
  • Boot volume encryption: Supported

Exam Tips for EBS Service Limitations

  • Most EBS limits are soft limits that can be increased via AWS Support
  • Multi-Attach is only supported on io1 and io2 volumes with Nitro instances
  • gp3 offers independent IOPS and throughput scaling up to 16,000 IOPS and 1,000 MiB/s
  • io2 Block Express provides the highest performance with up to 256,000 IOPS
  • EBS-optimized instances provide dedicated bandwidth for EBS traffic
  • Fast Snapshot Restore eliminates initialization latency but has cost implications
  • Encryption cannot be removed once enabled, but can be added via snapshot copy
