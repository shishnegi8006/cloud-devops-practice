EBS Snapshots

Backup, recovery, and snapshot management for EBS volumes

Snapshot Fundamentals

How Snapshots Work

Point-in-time: Captures volume state at specific moment

Incremental: Only changed blocks stored after first snapshot

Storage: Stored in Amazon S3 (not visible to users)

Snapshot States

Pending: Snapshot creation in progress

Completed: Snapshot ready for use

Error: Snapshot creation failed

Snapshot Creation & Management

Manual Snapshots

Create snapshots on-demand through console, CLI, or API.

Best Practices:

  • • Stop I/O before snapshot for consistency
  • • Use descriptive names and tags
  • • Document snapshot purpose
Use Case: Before major system changes

Automated Snapshots

Use Data Lifecycle Manager (DLM) for automated snapshot policies.

DLM Features:

  • • Schedule-based creation
  • • Automatic retention management
  • • Tag-based resource targeting
  • • Cross-region copy support
Benefit: Consistent backup without manual intervention

Snapshot Tagging

Organize and manage snapshots using comprehensive tagging strategies.

Recommended Tags:

  • • Environment (prod, dev, test)
  • • Application name
  • • Backup frequency
  • • Retention period
Benefit: Cost tracking and automated management

Cross-Region Capabilities

Cross-Region Copy

Copy snapshots to different AWS regions for disaster recovery.

Process:

  • • Select source snapshot
  • • Choose destination region
  • • Optionally change encryption
  • • Monitor copy progress
Use Case: Geographic redundancy and compliance

Encryption Handling

Manage encryption keys when copying snapshots across regions.

Options:

  • • Use destination region default key
  • • Specify custom KMS key
  • • Change encryption settings
Note: Encrypted snapshots remain encrypted

Automated Cross-Region

Use DLM policies to automatically copy snapshots across regions.

Configuration:

  • • Define target regions
  • • Set copy schedules
  • • Configure retention policies
Benefit: Automated DR without manual intervention

Fast Snapshot Restore (FSR)

How FSR Works

Pre-initialization: Snapshot data pre-loaded for instant access

No penalty: Full performance immediately after volume creation

AZ-specific: Enable FSR per Availability Zone

FSR Considerations

Cost: Additional charges per snapshot per AZ per hour

Time: Takes time to initialize (up to 60 minutes)

Limits: Maximum 50 FSR-enabled snapshots per region

Snapshot Sharing & Permissions

Cross-Account Sharing

Share snapshots with specific AWS accounts for collaboration.

Process:

  • • Modify snapshot permissions
  • • Add target account IDs
  • • Grant create volume permissions
Use Case: Development team collaboration

Public Snapshots

Make snapshots publicly available to all AWS users.

Considerations:

  • • Cannot make encrypted snapshots public
  • • Review data sensitivity carefully
  • • Consider compliance requirements
Use Case: AMI marketplace, open source projects

Permission Management

Control access using IAM policies and resource-based permissions.

IAM Actions:

  • • ec2:CreateSnapshot
  • • ec2:DeleteSnapshot
  • • ec2:ModifySnapshotAttribute
Best Practice: Principle of least privilege

Recovery Scenarios

Volume Recovery

Scenario:

EBS volume corruption or accidental deletion

Recovery Steps:

  1. 1. Identify appropriate snapshot
  2. 2. Create new volume from snapshot
  3. 3. Attach volume to EC2 instance
  4. 4. Mount and verify data integrity
RTO: 10-30 minutes | RPO: Last snapshot

Cross-Region Recovery

Scenario:

Regional disaster or service disruption

Recovery Steps:

  1. 1. Access cross-region snapshot copies
  2. 2. Create volumes in alternate region
  3. 3. Launch instances in DR region
  4. 4. Update DNS and routing
RTO: 1-4 hours | RPO: Cross-region copy frequency

Snapshot Cost Optimization

Lifecycle Management

Automatically delete old snapshots based on retention policies.

Strategy:

Keep daily for 7 days, weekly for 4 weeks, monthly for 12 months

Incremental Nature

Only changed blocks are stored, reducing storage costs.

Savings:

Significant cost reduction for frequent snapshots

Archive Storage

Move old snapshots to EBS Snapshot Archive for long-term retention.

Cost:

75% cheaper than standard snapshot storage

Monitoring Usage

Track snapshot storage costs and usage patterns.

Tools:

Cost Explorer, AWS Budgets, CloudWatch
