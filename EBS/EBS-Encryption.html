EBS Encryption

Secure your EBS volumes with encryption at rest and in transit

Encryption Fundamentals

How EBS Encryption Works

AES-256: Industry-standard encryption algorithm

Hardware-based: Encryption handled by EC2 instance hardware

No impact: Minimal performance overhead

What Gets Encrypted

Volume data: All data stored on the volume

Snapshots: Snapshots created from encrypted volumes

Data in transit: Data moving between EC2 and EBS

AWS KMS Integration

AWS Managed Keys

Use AWS-managed KMS keys for simple encryption setup.

Key ID:

aws/ebs (default EBS service key)

Benefits:

  • • No key management overhead
  • • Automatic key rotation
  • • No additional charges
Best For: Simple encryption requirements

Customer Managed Keys

Create and manage your own KMS keys for full control.

Features:

  • • Custom key policies
  • • Cross-account access
  • • Key rotation control
  • • Audit trail via CloudTrail
Best For: Compliance and governance requirements

Key Policies

Control access to encryption keys using IAM and key policies.

Permissions:

  • • kms:Encrypt/Decrypt
  • • kms:GenerateDataKey
  • • kms:CreateGrant
Principle: Least privilege access

Encryption Types

Encryption at Rest

Data is encrypted when stored on EBS volumes and in snapshots.

Implementation:

  • • Volume-level encryption
  • • Automatic snapshot encryption
  • • Metadata encryption

Key Management:

  • • Data keys generated per volume
  • • Keys encrypted with KMS key
  • • Keys stored with volume metadata

Encryption in Transit

Data is encrypted while moving between EC2 instances and EBS volumes.

Implementation:

  • • TLS encryption for API calls
  • • Hardware-accelerated encryption
  • • Automatic for encrypted volumes

Coverage:

  • • EC2 to EBS communication
  • • Snapshot operations
  • • Volume attachment/detachment

Encryption by Default

Account-Level Setting

Enable encryption by default for all new EBS volumes in your account.

Scope:

  • • Per-region setting
  • • Applies to all volume types
  • • Includes root volumes
Benefit: Ensures compliance without manual intervention

Default Key Selection

Choose which KMS key to use for encryption by default.

Options:

  • • AWS managed key (aws/ebs)
  • • Customer managed key
  • • Different keys per region
Flexibility: Can override on per-volume basis

Existing Volumes

Encryption by default doesn't affect existing unencrypted volumes.

Migration Options:

  • • Create encrypted snapshot
  • • Create new volume from snapshot
  • • Use AWS Systems Manager
Note: Cannot encrypt existing volumes in-place

Encryption Operations

Creating Encrypted Volumes

New volume: Enable encryption during creation

From snapshot: Inherits encryption from source

Copy encrypted: Can change encryption key

Migrating to Encryption

Step 1: Create snapshot of unencrypted volume

Step 2: Copy snapshot with encryption enabled

Step 3: Create encrypted volume from snapshot

Performance Considerations

Minimal Overhead

Hardware-accelerated encryption provides minimal performance impact.

Impact:

Typically less than 5% performance overhead

Hardware Acceleration

Modern EC2 instances include dedicated encryption hardware.

Support:

All current generation instance types

Monitoring

Monitor encryption performance using CloudWatch metrics.

Metrics:

Same metrics as unencrypted volumes

Compliance & Security

Compliance Standards

FIPS 140-2: Level 2 validated encryption

Common Criteria: EAL4+ certification

Standards: SOC, PCI DSS, HIPAA compliant

Security Best Practices

Key rotation: Enable automatic key rotation

Monitoring: Use CloudTrail for key usage audit

Access control: Implement least privilege policies
