Questions & Answers

CloudFront & Global Accelerator Practice

What is the main purpose of Amazon CloudFront?

Answer: Amazon CloudFront is a content delivery network (CDN) that delivers content to users with low latency and high transfer speeds by caching content at edge locations worldwide.

How does Global Accelerator differ from CloudFront?

Answer: Global Accelerator uses anycast IP addresses to route traffic to optimal AWS endpoints, while CloudFront caches content at edge locations. Global Accelerator works with TCP/UDP traffic, CloudFront primarily with HTTP/HTTPS.

What is Origin Access Identity (OAI)?

Answer: OAI is a special CloudFront user that restricts direct access to S3 bucket content, forcing users to access content through CloudFront for security and control.

When would you use signed URLs vs signed cookies?

Answer: Use signed URLs for individual files or when you want to restrict access to specific objects. Use signed cookies when providing access to multiple files or for streaming content.

What are cache behaviors in CloudFront?

Answer: Cache behaviors define how CloudFront handles requests for specific path patterns, including TTL settings, query string forwarding, and HTTP methods allowed.

What is Lambda@Edge used for?

Answer: Lambda@Edge runs code at AWS edge locations to customize content delivery, enable A/B testing, authentication, real-time image processing, and personalization.

How many edge locations does CloudFront have?

Answer: CloudFront has 400+ edge locations worldwide, strategically placed in major cities and regions to minimize latency for end users.

What are the benefits of using Global Accelerator?

Answer: Global Accelerator improves performance by up to 60%, provides static anycast IP addresses, automatic failover, and works with TCP/UDP traffic for better global application performance.
