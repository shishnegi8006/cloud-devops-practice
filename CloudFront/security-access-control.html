Security & Access Control

Protection Mechanisms

Origin Access Identity (OAI)

  • Restricts direct access to S3 bucket content
  • Forces users to access content through CloudFront
  • Special CloudFront user for S3 bucket access
  • Prevents bypassing CloudFront security features
  • Works only with S3 origins, not custom origins

Signed URLs & Cookies

  • Control access to premium or private content
  • Time-based access with expiration dates
  • IP address restrictions for additional security
  • Signed URLs for individual files
  • Signed cookies for multiple files or streaming

Exam Tips for Security & Access Control

  • OAI prevents direct S3 access, forcing traffic through CloudFront
  • Signed URLs/cookies provide time-limited access to private content
  • WAF integration protects against common web attacks
  • SSL/TLS encryption secures data in transit
