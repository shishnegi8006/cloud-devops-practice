CloudFront Fundamentals

Content Delivery Network Basics

CDN Concepts

  • Content Delivery Network distributes content globally
  • Reduces latency by serving content from nearest edge location
  • Caches static and dynamic content at edge locations
  • Improves website performance and user experience
  • Reduces load on origin servers
  • Provides DDoS protection and security features

Edge Locations

  • 400+ edge locations worldwide
  • Located in major cities and regions
  • Cache content closer to end users
  • Regional Edge Caches for less popular content
  • Automatic failover between edge locations
  • Real-time metrics and monitoring

Distribution Types

  • Web Distribution: Static and dynamic web content
  • RTMP Distribution: Media streaming (deprecated)
  • Supports HTTP/HTTPS protocols
  • Multiple origins per distribution
  • Custom error pages and redirects
  • Geographic restrictions and access control

Origin Types

  • S3 Bucket: Static website hosting and files
  • Custom Origin: EC2, ELB, on-premises servers
  • MediaStore: Video streaming content
  • MediaPackage: Live and on-demand video
  • Origin failover and redundancy
  • Custom headers and authentication

Key Features

  • Global content delivery with low latency
  • SSL/TLS encryption and custom certificates
  • Real-time logs and detailed analytics
  • Integration with AWS WAF for security
  • Lambda@Edge for edge computing
  • HTTP/2 and IPv6 support

Pricing Model

  • Pay for data transfer out from edge locations
  • HTTP/HTTPS requests pricing
  • Regional pricing variations
  • Free tier: 50 GB data transfer, 2M requests
  • Price classes for cost optimization
  • No upfront costs or minimum commitments

Exam Tips for CloudFront Fundamentals

  • CloudFront is a global CDN service with 400+ edge locations
  • Edge locations cache content to reduce latency and improve performance
  • Supports both S3 and custom origins (EC2, ELB, on-premises)
  • Web distributions handle HTTP/HTTPS traffic for websites and APIs
  • Regional Edge Caches provide additional caching layer for less popular content
  • Integrates with other AWS services like WAF, Lambda@Edge, and Route 53
