Storage Options Overview

Network-attached persistent storage

Managed NFS file system

EBS Volume Types

Type Use Case IOPS Throughput

gp3: Latest generation, independent IOPS and throughput provisioning
io2: Higher durability (99.999%), sub-millisecond latency
st1: Big data, data warehouses, log processing

EBS Features

Snapshots

  • • Point-in-time backups stored in S3
  • • Incremental - only changed blocks
  • • Can copy across regions
  • • Create AMIs from snapshots

Encryption

  • • Data at rest and in transit encryption
  • • AWS KMS key management
  • • Minimal impact on latency
  • • Encrypted snapshots and AMIs

Multi-Attach

  • • io1/io2 volumes only
  • • Up to 16 instances (same AZ)
  • • Cluster-aware file systems required
  • • Higher availability applications
# Create encrypted EBS volume
aws ec2 create-volume \
  --size 100 \
  --volume-type gp3 \
  --availability-zone us-east-1a \
  --encrypted \
  --iops 4000 \
  --throughput 250

Ephemeral Storage

  • • Data lost on instance stop/termination
  • • Data persists during reboot
  • • Cannot be detached/reattached
  • • No additional cost

Use Cases

  • • Temporary storage, caches, buffers
  • • High-performance databases
  • • Distributed file systems
  • • Content that's replicated across fleet

Instance Types with Instance Store

C5d, M5d, R5d, I3, I3en, D2, D3, H1, and more

EFS Integration

Elastic File System provides scalable NFS for EC2 instances.

  • Multi-AZ: Mount from multiple AZs simultaneously
  • POSIX-compliant: Standard file system interface
  • Scalable: Petabyte-scale, pay for what you use
  • Performance modes: General Purpose vs Max I/O

Performance Classes

  • • General Purpose: Lowest latency per operation
  • • Max I/O: Higher aggregate throughput and IOPS

Throughput Modes

  • • Bursting: Throughput scales with file system size
  • • Provisioned: Fixed throughput independent of size
# Mount EFS on EC2
sudo mount -t efs \
  fs-12345678:/ /mnt/efs

# Using EFS utils
sudo mount -t efs \
  -o tls fs-12345678:/ /mnt/efs

Performance Optimization

RAID Configurations

  • • RAID 0: Striping for increased performance
  • • RAID 1: Mirroring for increased fault tolerance
  • • RAID 5/6: Not recommended on EBS

EBS Optimization

  • • Enable EBS-optimized instances
  • • Use appropriate volume types
  • • Pre-warm volumes from snapshots
  • • Monitor CloudWatch metrics

Key Metrics

  • • VolumeReadOps/VolumeWriteOps: IOPS
  • • VolumeReadBytes/VolumeWriteBytes: Throughput
  • • VolumeTotalReadTime/VolumeTotalWriteTime: Latency
  • • VolumeQueueLength: Queue depth

Backup & Recovery

EBS Snapshots

  • • Incremental backups to S3
  • • Can be automated with Data Lifecycle Manager
  • • Cross-region copy for disaster recovery
  • • Fast Snapshot Restore for quick recovery

AWS Backup

  • • Centralized backup across AWS services
  • • Policy-based backup plans
  • • Cross-region and cross-account backup
  • • Compliance and audit reporting

Best Practices

  • • Regular automated snapshots
  • • Test restore procedures
  • • Tag snapshots for lifecycle management
  • • Monitor backup completion
# Create snapshot
aws ec2 create-snapshot \
  --volume-id vol-12345678 \
  --description "Daily backup"

# Copy to another region
aws ec2 copy-snapshot \
  --source-region us-east-1 \
  --source-snapshot-id snap-12345678 \
  --destination-region us-west-2
