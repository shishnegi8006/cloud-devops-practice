EC2 Backup & Restore

Complete guide to data protection and disaster recovery strategies

EBS Snapshots

Point-in-Time Snapshots

Create incremental backups of EBS volumes stored in Amazon S3.

Use Case:

Daily backups of production database volumes

Benefit: Only changed blocks are stored, reducing costs

Automated Snapshots

Schedule automatic snapshots using Data Lifecycle Manager (DLM).

Use Case:

Automated daily backups with 30-day retention

Feature: Tag-based policies and cross-region copying

Cross-Region Snapshots

Copy snapshots to different regions for disaster recovery.

Use Case:

Disaster recovery with geographic separation

Benefit: Protection against regional failures

Encrypted Snapshots

Snapshots inherit encryption from source volumes automatically.

Use Case:

Secure backups of sensitive data with KMS encryption

Security: Data encrypted at rest and in transit

Fast Snapshot Restore

Enable FSR for instant volume creation from snapshots.

Use Case:

Quick recovery for critical production systems

Performance: No initialization penalty on first access

Snapshot Sharing

Share snapshots across AWS accounts or make them public.

Use Case:

Share golden images across development accounts

Control: Granular permissions and access control

AMI (Amazon Machine Image) Backups

Complete Instance Images

Create full system images including OS, applications, and configurations.

Use Case:

Golden images for auto scaling groups

Components: Root volume + additional EBS volumes

Launch Templates

Use AMIs in launch templates for consistent deployments.

Use Case:

Standardized instance configurations across environments

Benefit: Version control for infrastructure

AMI Copying

Copy AMIs across regions and accounts for distribution.

Use Case:

Multi-region deployments and disaster recovery

Feature: Encryption and permission management

Disaster Recovery Scenarios

Instance Failure Recovery

Scenario:

EC2 instance becomes unresponsive or corrupted

Recovery Steps:

  1. 1. Stop the failed instance
  2. 2. Create new volume from latest snapshot
  3. 3. Attach volume to new instance
  4. 4. Update DNS/load balancer
RTO: 15-30 minutes | RPO: Last snapshot

Regional Disaster Recovery

Scenario:

Entire AWS region becomes unavailable

Recovery Steps:

  1. 1. Switch to secondary region
  2. 2. Launch instances from cross-region AMIs
  3. 3. Restore data from cross-region snapshots
  4. 4. Update Route 53 DNS records
RTO: 1-4 hours | RPO: Last cross-region sync

Cost Optimization for Backups

Lifecycle Policies

Automatically delete old snapshots based on age.

Example:

Keep daily for 7 days, weekly for 4 weeks

Incremental Backups

EBS snapshots only store changed blocks.

Savings:

Up to 90% storage cost reduction

Archive Storage

Move old snapshots to EBS Snapshot Archive.

Cost:

75% cheaper than standard snapshots

Tag-Based Billing

Track backup costs by environment or project.

Benefit:

Cost allocation and optimization
