EC2 Fundamentals

Elastic Compute Cloud Core Concepts

Instance Families

T/M General Purpose: Balanced resources for web servers.
C Compute Optimized: High performance processors for batch processing, gaming.
R Memory Optimized: RAM intensive for databases, caching.
I/D Storage Optimized: High I/O for NoSQL DBs, data warehousing.
P/G Accelerated: GPU for machine learning, graphics.

Bootstrapping

  • User Data: Script run once at first launch with root privileges.
  • IMDSv2: Securely access instance metadata (IP, AMI ID) from within the OS.
#!/bin/bash
yum update -y
yum install -y httpd
systemctl start httpd
systemctl enable httpd
echo "Hello World from $(hostname -f)" > /var/www/html/index.html

Security Groups

A virtual firewall that controls inbound and outbound traffic for your instances.

  • Stateful: If you send a request out, the response traffic is automatically allowed back in.
  • Allow Rules Only: You cannot explicitly deny traffic (use NACLs for deny rules).
  • Scope: Applied at the instance level (Network Interface), not the subnet level.

Storage Options

EBS (Elastic Block Store)

Network drive. Persistent data. Can be detached and re-attached to other instances in the same AZ.

Instance Store

Physically attached to the host. Ephemeral (data is LOST if instance stops). Extremely high I/O performance.

AMI (Amazon Machine Image)

A template for the root volume (OS, app server, applications) used to launch instances.
