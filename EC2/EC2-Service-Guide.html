EC2 Instance Types

General Purpose (T, M, A)

Balanced compute, memory, and networking for diverse workloads.

Use Case:

Web servers, small databases, development environments

Example: t3.medium for small web applications

Compute Optimized (C)

High-performance processors for compute-intensive tasks.

Use Case:

Scientific computing, gaming servers, high-performance web servers

Example: c5.xlarge for CPU-intensive applications

Memory Optimized (R, X, Z)

High memory-to-CPU ratio for memory-intensive applications.

Use Case:

In-memory databases, real-time analytics, caching

Example: r5.large for Redis or Memcached

Storage Optimized (I, D, H)

High sequential read/write access to large datasets.

Use Case:

NoSQL databases, data warehousing, distributed file systems

Example: i3.large for MongoDB or Cassandra

Accelerated Computing (P, G, F)

Hardware accelerators like GPUs for specialized workloads.

Use Case:

Machine learning, 3D rendering, video processing

Example: p3.2xlarge for deep learning training

EC2 Pricing Models

On-Demand

Pay by the hour/second with no long-term commitments.

Best For:

Testing, development, unpredictable workloads

Reserved Instances

1-3 year commitment for up to 75% discount.

Best For:

Steady-state production workloads

Spot Instances

Bid on spare capacity for up to 90% discount.

Best For:

Batch processing, fault-tolerant workloads

Dedicated Hosts

Physical servers dedicated to your use.

Best For:

Compliance, licensing requirements

Core EC2 Features

Auto Scaling

Automatically adjust capacity based on demand.

Use Case:

Handle traffic spikes during Black Friday sales

Benefit: Cost optimization and high availability

Load Balancing

Distribute traffic across multiple instances.

Use Case:

Ensure website stays online if one server fails

Types: Application, Network, Gateway Load Balancers

Security Groups

Virtual firewalls controlling inbound/outbound traffic.

Use Case:

Allow web traffic (port 80/443) but block SSH from internet

Rule: Stateful - return traffic automatically allowed

EBS Volumes

Persistent block storage that can be attached to instances.

Use Case:

Database storage that persists even if instance terminates

Types: gp3, io2, st1, sc1 for different performance needs

AMI (Amazon Machine Images)

Pre-configured templates for launching instances.

Use Case:

Create custom AMI with your application pre-installed

Benefit: Faster deployment and consistent environments

Key Pairs

Secure login credentials for SSH access to EC2 instances.

Use Case:

Secure SSH access to Linux instances for administration

Note: Store private key securely, cannot be recovered if lost

Network Interfaces

Virtual network cards that can be attached to EC2 instances.

Use Case:

Dual-homed instances, network failover, licensing requirements

Types: Primary ENI (eth0), Secondary ENI, Elastic Network Interface
