CloudWatch Metrics

AWS CloudWatch provides comprehensive monitoring for EC2 instances with both basic and detailed metrics.

Basic Monitoring (Free)

  • • 5-minute intervals
  • • 7 pre-selected metrics
  • • Enabled by default
  • • No additional cost

Detailed Monitoring

  • • 1-minute intervals
  • • All available metrics
  • • Additional cost per instance
  • • Better for auto scaling

# Enable detailed monitoring
aws ec2 monitor-instances \
  --instance-ids i-12345678

# Get metric statistics
aws cloudwatch get-metric-statistics \
  --namespace AWS/EC2 \
  --metric-name CPUUtilization \
  --dimensions Name=InstanceId,Value=i-12345678

CloudWatch Agent

Enhanced monitoring with custom metrics and log collection from EC2 instances.

  • System Metrics: Memory, disk space, swap utilization
  • Process Metrics: CPU and memory usage by process
  • Log Collection: Application and system logs
  • Custom Metrics: Application-specific metrics

Memory Metrics

  • • MemoryUtilization
  • • MemoryUsed
  • • MemoryAvailable
  • • SwapUtilization

Disk Metrics

  • • DiskSpaceUtilization
  • • DiskSpaceUsed
  • • DiskSpaceAvailable
  • • InodeUtilization
# Install CloudWatch Agent
wget https://s3.amazonaws.com/amazoncloudwatch-agent/amazon_linux/amd64/latest/amazon-cloudwatch-agent.rpm
sudo rpm -U ./amazon-cloudwatch-agent.rpm

# Configure and start
sudo /opt/aws/amazon-cloudwatch-agent/bin/amazon-cloudwatch-agent-config-wizard
sudo systemctl start amazon-cloudwatch-agent

Status Checks

Automated checks that monitor the health of your EC2 instances and underlying infrastructure.

System Status Checks

  • • AWS infrastructure issues
  • • Network connectivity problems
  • • Host hardware/software issues
  • • Resolution: Stop and start instance

Instance Status Checks

  • • Instance configuration issues
  • • Kernel problems
  • • Network configuration
  • • Resolution: Reboot or fix configuration

Status Check Results

Passed - All checks successful
Failed - Issues detected
Initializing - Checks in progress

CloudWatch Alarms

Proactive monitoring with automated actions based on metric thresholds.

Critical Alarm

CPU > 90% for 2 consecutive periods

Warning Alarm

Memory > 80% for 5 minutes

Info Alarm

Network throughput anomaly detected

Alarm Actions

  • • Send SNS notifications
  • • Auto Scaling actions
  • • EC2 actions (stop, terminate, reboot)
  • • Systems Manager actions
# Create CPU alarm
aws cloudwatch put-metric-alarm \
  --alarm-name "High-CPU-Utilization" \
  --alarm-description "CPU > 80%" \
  --metric-name CPUUtilization \
  --namespace AWS/EC2 \
  --statistic Average \
  --period 300 \
  --threshold 80 \
  --comparison-operator GreaterThanThreshold

Logging

Centralized log management with CloudWatch Logs for troubleshooting and analysis.

CloudWatch Logs

  • • Centralized log storage and analysis
  • • Real-time log streaming
  • • Log retention policies
  • • Integration with other AWS services

Log Insights

  • • Interactive log analytics
  • • Query and visualize log data
  • • Pattern detection
  • • Performance analysis

Common Log Sources

  • /var/log/messages (system logs)
  • /var/log/secure (authentication logs)
  • Application logs
  • Web server logs (Apache, Nginx)
# CloudWatch Logs Insights query

Troubleshooting Guide

Instance Won't Start

  • • Check instance limits and quotas
  • • Verify AMI availability in region
  • • Check security group rules
  • • Review subnet configuration

Connectivity Issues

  • • Verify security group inbound rules
  • • Check NACL rules
  • • Confirm route table configuration
  • • Test with EC2 Instance Connect

Performance Issues

  • • Monitor CPU, memory, and disk metrics
  • • Check for CPU credit balance (T instances)
  • • Review EBS volume performance
  • • Analyze network utilization

Storage Issues

  • • Check disk space utilization
  • • Monitor EBS volume metrics
  • • Verify file system health
  • • Review backup and snapshot status

Best Practices

  • • Enable detailed monitoring for critical instances
  • • Set up comprehensive alarms
  • • Use Systems Manager for patch management
  • • Implement proper tagging strategy
