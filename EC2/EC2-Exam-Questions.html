EC2 Exam Questions

Practice Questions with Detailed Answers

Question 1

Which EC2 instance type is best suited for high-performance databases and in-memory analytics?

A) T3 (General Purpose)
B) C5 (Compute Optimized)
C) R5 (Memory Optimized)
D) I3 (Storage Optimized)

Answer: C) R5 (Memory Optimized)

R5 instances are memory optimized and provide high memory-to-vCPU ratios, making them ideal for memory-intensive applications like high-performance databases, in-memory analytics, and real-time big data analytics.

Question 2

What happens to data stored on instance store volumes when an EC2 instance is stopped?

A) Data is preserved and available when the instance restarts
B) Data is permanently lost
C) Data is automatically backed up to S3
D) Data is moved to EBS volumes

Answer: B) Data is permanently lost

Instance store volumes provide temporary block-level storage. Data is lost when the instance stops, terminates, or fails. This is why instance store is called "ephemeral storage" and should only be used for temporary data, caches, or data that can be recreated.

Question 3

Which placement group type provides the lowest network latency between instances?

A) Cluster placement group
B) Spread placement group
C) Partition placement group
D) Default placement

Answer: A) Cluster placement group

Cluster placement groups pack instances close together in a single Availability Zone, providing the lowest latency and highest network throughput. This is ideal for HPC applications that require tight coupling between instances.

Question 4

What is the maximum number of instances you can have in a spread placement group per Availability Zone?

A) 5 instances
B) 7 instances
C) 10 instances
D) 20 instances

Answer: B) 7 instances

Spread placement groups can have a maximum of 7 running instances per Availability Zone. This limitation ensures that each instance runs on distinct underlying hardware, providing maximum isolation for critical applications.

Question 5

Which Auto Scaling policy type automatically adjusts capacity to maintain a target metric value?

A) Simple Scaling
B) Step Scaling
C) Target Tracking Scaling
D) Scheduled Scaling

Answer: C) Target Tracking Scaling

Target Tracking Scaling automatically adjusts the number of instances to maintain a specified target value for a metric (like CPU utilization at 70%). It's the easiest scaling policy to set up and works like a thermostat.

Question 6

What is the recommended approach for providing credentials to applications running on EC2 instances?

A) Hard-code AWS access keys in the application
B) Store credentials in environment variables
C) Use IAM roles attached to EC2 instances
D) Store credentials in a configuration file

Answer: C) Use IAM roles attached to EC2 instances

IAM roles provide temporary, automatically rotating credentials to EC2 instances. This is the most secure method as it eliminates the need to manage long-term access keys and follows AWS security best practices.

Question 7

Which EC2 pricing model offers the highest discount but can be interrupted with a 2-minute warning?

A) On-Demand Instances
B) Reserved Instances
C) Spot Instances
D) Dedicated Hosts

Answer: C) Spot Instances

Spot Instances can provide up to 90% discount compared to On-Demand prices but can be interrupted by AWS with a 2-minute warning when capacity is needed. They're ideal for fault-tolerant, flexible workloads like batch processing, data analysis, and stateless applications.
