Network Limitations

Security Groups per Instance

Maximum of 5 security groups per network interface. Each instance can have multiple network interfaces.

Rules per Security Group

Maximum of 60 inbound and 60 outbound rules per security group.

Elastic IP Addresses

Default limit of 5 Elastic IPs per region. Can be increased via support request.

Network Interfaces per Instance

Varies by instance type. Small instances: 2-3, Large instances: 8-15 network interfaces.

Regional & Availability Zone Constraints

Instance Mobility

Instances cannot be moved between regions. Can only be moved between AZs by creating AMI and launching new instance.

EBS Volume Attachment

EBS volumes can only be attached to instances in the same Availability Zone.

Instance Store Persistence

Instance store data is lost permanently when instance stops, terminates, or fails. Cannot be detached/reattached.

Important Considerations

Soft Limits: Most EC2 limits are soft limits that can be increased by submitting a support case to AWS.

Spot Instance Interruption: Spot instances can be interrupted with only 2-minute warning when capacity is needed.

Instance Type Availability: Not all instance types are available in all AZs. Check availability before deployment.
