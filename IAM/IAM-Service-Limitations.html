Access Key & Authentication Limits

Access Keys per User

Maximum of 2 access keys per IAM user. Cannot create more than 2 active access keys simultaneously.

MFA Devices per User

Maximum of 8 MFA devices per IAM user (including virtual, hardware, and U2F devices).

Assume Role Session Duration

Minimum: 15 minutes, Maximum: 12 hours (can be configured per role).

Regional & Global Constraints

Global Service

IAM is a global service - users, groups, roles, and policies are available across all regions but managed from a single global namespace.

Cross-Account Limitations

Cannot directly share IAM users across accounts. Must use cross-account roles or federated access.

Root User Restrictions

Root user cannot be deleted, cannot have policies attached directly, and has unrestricted access that cannot be limited by SCPs.

Important Considerations

Soft Limits: Many IAM limits can be increased by contacting AWS Support, but some are hard limits.

Policy Evaluation: Complex policies with many conditions can impact performance. Keep policies simple and specific.

Eventually Consistent: IAM changes are eventually consistent. Allow time for propagation across all regions.
