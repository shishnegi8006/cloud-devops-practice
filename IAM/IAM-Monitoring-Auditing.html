CloudTrail, Access Analyzer, and Compliance Tools

CloudTrail Integration

AWS CloudTrail provides comprehensive logging of all IAM API calls and user activities.

API Call Logging

  • • All IAM API calls logged
  • • User identity and source IP
  • • Request parameters and responses
  • • Timestamp and region information

Authentication Events

  • • Console sign-in attempts
  • • MFA authentication events
  • • AssumeRole operations
  • • Failed authentication attempts

Credential Usage

  • • Access key usage tracking
  • • Temporary credential issuance
  • • Service-to-service calls
  • • Cross-account access patterns
Key CloudTrail Events
ConsoleLogin
User Sign-in
AssumeRole
Role Assumption
CreateUser
User Creation
AttachPolicy
Policy Changes
# Query CloudTrail logs
aws logs filter-log-events \
  --log-group-name CloudTrail/IAMEvents \
  --filter-pattern "{ $.eventName = AssumeRole }" \
  --start-time 1640995200000

IAM Access Analyzer

Analyze resource policies and identify unintended access to your resources from external entities.

  • External Access: Identify resources accessible from outside your account
  • Policy Validation: Check policies for syntax and logic errors
  • Unused Access: Find permissions that haven't been used
  • Policy Generation: Generate policies based on access patterns

Finding Types

HIGH Public read access to S3 bucket
MEDIUM Cross-account role trust
LOW Unused IAM role

Analyzer Types

  • • Account: Analyze resources in your account
  • • Organization: Analyze across AWS Organizations
  • • Custom: Define your own zone of trust
# Create Access Analyzer
aws accessanalyzer create-analyzer \
  --analyzer-name MyAccountAnalyzer \
  --type ACCOUNT

# List findings
aws accessanalyzer list-findings \
  --analyzer-arn arn:aws:access-analyzer:us-east-1:123456789012:analyzer/MyAccountAnalyzer

Credential Reports

Generate comprehensive reports about user credentials and their usage patterns.

Report Contents

  • • User creation and last activity dates
  • • Password and access key status
  • • MFA device assignments
  • • Certificate and signing certificate info

Usage Tracking

  • • Last console login timestamp
  • • Access key last used date and service
  • • Password last changed date
  • • Identify inactive users and credentials

Common Use Cases

  • Compliance audits and reporting
  • Identify unused or stale credentials
  • Password policy compliance
  • MFA adoption tracking
# Generate credential report
aws iam generate-credential-report

# Download credential report
aws iam get-credential-report \
  --output text \
  --query 'Content' | base64 -d > credential-report.csv

# Deploy Config rule
aws configservice put-config-rule \
  --config-rule '{
    "ConfigRuleName": "iam-user-mfa-enabled",
    "Source": {
      "Owner": "AWS",
      "SourceIdentifier": "IAM_USER_MFA_ENABLED"
    }
  }'

Alerting & Notifications

Set up proactive monitoring and alerting for IAM security events and policy violations.

Critical Alerts
  • • Root account usage
  • • New IAM user creation
  • • Policy changes with admin permissions
  • • Multiple failed login attempts
Warning Alerts
  • • Unusual access patterns
  • • Cross-account role assumptions
  • • Access key rotation overdue
  • • MFA device changes
Informational Alerts
  • • New resource access
  • • Policy validation results
  • • Credential report generation
  • • Access Analyzer findings

Alert Mechanisms

  • • CloudWatch Alarms and SNS
  • • EventBridge rules and targets
  • • Lambda functions for custom logic
  • • Third-party SIEM integration
# CloudWatch alarm for root usage
aws cloudwatch put-metric-alarm \
  --alarm-name "RootAccountUsage" \
  --alarm-description "Root account used" \
  --metric-name RootAccountUsageEventCount \
  --namespace CloudWatchLogMetrics \
  --statistic Sum \
  --period 300 \
  --threshold 1 \
  --comparison-operator GreaterThanOrEqualToThreshold
