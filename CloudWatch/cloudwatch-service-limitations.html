CloudWatch Service Limitations

Quotas, Limits & Regional Constraints

Service Limits Overview

AWS CloudWatch has various service limits and quotas that you should be aware of when designing monitoring solutions. These limits help ensure service stability and fair resource allocation.

Types of Limits

  • • Hard limits (cannot be increased)
  • • Soft limits (can request increases)
  • • Regional limits
  • • Account-level limits

Impact Areas

  • • Metric publishing
  • • Log ingestion
  • • API call rates
  • • Storage capacity

Mitigation Strategies

Request Limit Increases

For adjustable limits, submit a support case to request increases. Provide justification and expected usage patterns.

Batch Operations

Use batch APIs like PutMetricData to send multiple metrics in a single request, reducing API call overhead.

Implement Backoff

Use exponential backoff and jitter when encountering rate limits to avoid overwhelming the service.

Optimize Metrics

Reduce metric cardinality by carefully choosing dimensions. Aggregate metrics at the application level when possible.

Key Takeaways

  • CloudWatch has various service limits that affect metrics, logs, alarms, and API operations
  • Some limits are adjustable through support requests, while others are hard limits
  • Data retention periods are fixed and cannot be extended beyond the default limits
  • API rate limits can be increased, but proper batching and backoff strategies are essential
  • Understanding limits is crucial for designing scalable monitoring solutions
