A company wants to monitor the memory utilization of their EC2 instances. They notice that CloudWatch is not showing memory metrics by default. What is the best solution to monitor memory utilization?

A. Enable detailed monitoring for EC2 instances
Incorrect: Detailed monitoring only provides higher frequency for existing metrics, not additional metrics like memory utilization.
B. Install and configure the CloudWatch Agent on EC2 instances
Correct: The CloudWatch Agent can collect system-level metrics including memory utilization, disk usage, and other custom metrics not available by default.
C. Use AWS Systems Manager to collect memory metrics
Incorrect: While Systems Manager can collect some metrics, CloudWatch Agent is the recommended solution for detailed system metrics.
D. Create custom metrics using the CloudWatch API
Incorrect: While possible, this would require custom scripting and is not the best solution when CloudWatch Agent is available.
2

An application generates high-frequency metrics that need to be monitored in near real-time. The default 5-minute metric resolution is not sufficient. What should be implemented?

A. Enable detailed monitoring for all AWS services
Incorrect: Detailed monitoring only provides 1-minute resolution, which may still not be sufficient for high-frequency monitoring.
B. Publish high-resolution custom metrics with 1-second granularity
Correct: High-resolution metrics can be published with up to 1-second granularity, enabling near real-time monitoring for critical applications.
C. Use CloudWatch Logs for real-time monitoring
Incorrect: CloudWatch Logs is for log data, not metrics. While you can create metrics from logs, it's not the best solution for high-frequency metrics.
D. Implement a third-party monitoring solution
Incorrect: CloudWatch supports high-resolution metrics natively, so there's no need for a third-party solution.
3

A company wants to automatically scale their Auto Scaling Group when CPU utilization exceeds 70% for 2 consecutive periods of 5 minutes. How should this be configured?

A. Create an alarm with threshold 70%, period 5 minutes, evaluation periods 1
Incorrect: This would trigger after only 1 period, not 2 consecutive periods as required.
B. Create an alarm with threshold 70%, period 5 minutes, evaluation periods 2, datapoints to alarm 2
Correct: This configuration will trigger the alarm only when CPU exceeds 70% for 2 out of 2 consecutive 5-minute periods.
C. Create an alarm with threshold 70%, period 10 minutes, evaluation periods 1
Incorrect: This uses a 10-minute period instead of the required 5-minute periods.
D. Create an alarm with threshold 70%, period 5 minutes, evaluation periods 2, datapoints to alarm 1
Incorrect: This would trigger if only 1 out of 2 periods breaches the threshold, not requiring consecutive breaches.
4

A development team needs to analyze application logs to identify error patterns and create metrics from log data. Which CloudWatch feature should they use?

A. CloudWatch Dashboards
Incorrect: Dashboards are for visualization, not for analyzing log data or creating metrics from logs.
B. CloudWatch Alarms
Incorrect: Alarms monitor metrics and trigger actions, but don't analyze log data or create metrics from logs.
C. CloudWatch Logs Insights and Metric Filters
Correct: Logs Insights provides powerful querying capabilities for log analysis, while Metric Filters can create metrics from log data patterns.
D. CloudWatch Events
Incorrect: CloudWatch Events (now EventBridge) is for event-driven architectures, not log analysis.
5

A company has multiple alarms that often trigger false positives due to temporary spikes. They want to create a more sophisticated alerting system that only triggers when multiple conditions are met. What should they implement?

A. Increase the threshold values for all alarms
Incorrect: Simply increasing thresholds might miss legitimate issues and doesn't address the need for multiple conditions.
B. Use longer evaluation periods for all alarms
Incorrect: Longer periods might reduce false positives but delay legitimate alerts and don't provide complex logic.
C. Create composite alarms using Boolean logic (AND, OR, NOT)
Correct: Composite alarms allow combining multiple alarms with Boolean logic to create sophisticated alerting rules and reduce false positives.
D. Disable all alarms and rely on manual monitoring
Incorrect: Manual monitoring is not scalable and defeats the purpose of automated alerting.
6

An organization needs to retain CloudWatch Logs data for 7 years for compliance purposes, but the maximum retention period in CloudWatch Logs is 10 years. What is the most cost-effective solution?

A. Set CloudWatch Logs retention to 10 years
Incorrect: While this meets the requirement, it's not cost-effective as CloudWatch Logs storage is more expensive than S3 for long-term retention.
B. Export logs to S3 and use S3 lifecycle policies for long-term archival
Correct: Exporting to S3 and using lifecycle policies (Glacier, Deep Archive) is the most cost-effective solution for long-term log retention.
C. Use multiple CloudWatch Logs accounts to extend retention
Incorrect: This doesn't extend retention beyond the service limits and adds unnecessary complexity.
D. Store logs in EBS volumes attached to EC2 instances
Incorrect: EBS storage is more expensive than S3 for archival and doesn't provide the durability and lifecycle management features needed.

Study Tips for CloudWatch

Understand Metric Types

Know the difference between default AWS metrics and custom metrics. Understand when to use high-resolution metrics and their cost implications.

Master Alarm Configuration

Practice configuring alarms with different evaluation periods, datapoints to alarm, and missing data treatment options.

Know Integration Points

Understand how CloudWatch integrates with Auto Scaling, SNS, Lambda, and other AWS services for automated responses.

Practice Log Analysis

Get hands-on experience with CloudWatch Logs Insights queries and metric filters to extract meaningful data from logs.
