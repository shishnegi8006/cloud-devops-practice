CloudWatch Logs Overview

CloudWatch Logs enables you to centralize logs from all your systems, applications, and AWS services. You can monitor, store, and access log files from EC2 instances, CloudTrail, Route 53, and other sources.

Core Concepts

Log Groups

Log groups are containers for log streams that share the same retention, monitoring, and access control settings.

Characteristics

  • • Logical grouping of log streams
  • • Shared retention policy
  • • Common access control
  • • Metric filters and subscriptions

Naming Examples

  • • /aws/lambda/function-name
  • • /aws/apigateway/api-name
  • • /var/log/messages
  • • /application/web-server

Log Streams

Log streams are sequences of log events that share the same source. Each log stream belongs to one log group.

Properties

  • • Sequence of log events
  • • Single source (instance, container)
  • • Chronologically ordered
  • • Unique within log group

Examples

  • • i-1234567890abcdef0
  • • container-id-12345
  • • 2024/01/15/[$LATEST]abc123
  • • web-server-01

Log Events

Log events are individual log entries with a timestamp and message. They are the basic unit of log data in CloudWatch Logs.

Event Structure

timestamp: 1642248000000
message: "2024-01-15 10:00:00 INFO User login successful"

Log Sources

AWS Services

  • • Lambda: Function execution logs
  • • API Gateway: Access and execution logs
  • • CloudTrail: API call logs
  • • VPC Flow Logs: Network traffic logs
  • • ECS: Container logs
  • • EKS: Kubernetes cluster logs
  • • RDS: Database logs

Custom Applications

  • • EC2 Instances: Application and system logs
  • • On-premises: Hybrid cloud logging
  • • Containers: Docker and Kubernetes logs
  • • Mobile Apps: Client-side logging
  • • IoT Devices: Device telemetry
  • • Web Applications: Access and error logs

Log Retention Policies

Log retention policies determine how long log events are stored in CloudWatch Logs. You can set different retention periods for different log groups.

Retention Options

• 1 day to 10 years
• Never expire (indefinite)
• Common periods: 7, 30, 90, 365 days
• Configurable per log group

Cost Considerations

• Longer retention = higher costs
• Archive to S3 for long-term storage
• Use lifecycle policies
• Monitor storage usage

CloudWatch Logs Insights

CloudWatch Logs Insights enables you to interactively search and analyze log data using a purpose-built query language.

Query Capabilities

  • • Filter and search log events
  • • Extract and analyze fields
  • • Aggregate and visualize data
  • • Time-based analysis

Example Query

fields @timestamp, @message
| filter @message like /ERROR/
| stats count() by bin(5m)
| sort @timestamp desc

Best Practices

Log Structure

Use structured logging (JSON) for better searchability and analysis. Include consistent fields like timestamp, level, and correlation IDs.

Retention Strategy

Set appropriate retention periods based on compliance requirements and cost considerations. Archive old logs to S3 for long-term storage.

Security

Encrypt logs at rest and in transit. Use IAM policies to control access. Avoid logging sensitive information like passwords or PII.

Cost Optimization

Monitor log ingestion and storage costs. Use log sampling for high-volume applications. Implement log level filtering.

Key Takeaways

  • CloudWatch Logs provides centralized log management for AWS and custom applications
  • Log groups contain log streams, which contain individual log events
  • Logs Insights enables powerful querying and analysis of log data
  • Subscription filters enable real-time log processing and streaming
  • Proper retention policies and structured logging are essential for cost-effective log management
