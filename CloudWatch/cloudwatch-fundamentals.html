CloudWatch Fundamentals

Core Concepts and Monitoring Basics

What is Amazon CloudWatch?

Amazon CloudWatch is a monitoring and observability service built for DevOps engineers, developers, site reliability engineers (SREs), and IT managers. CloudWatch provides data and actionable insights to monitor applications, respond to system-wide performance changes, optimize resource utilization, and get a unified view of operational health.

Key Benefits

  • • Real-time monitoring
  • • Automated actions
  • • Unified operational view
  • • Cost optimization

Use Cases

  • • Application monitoring
  • • Infrastructure monitoring
  • • Log analysis
  • • Performance optimization

Core Components

Metrics

Metrics are data about the performance of your systems. CloudWatch receives metrics from AWS services automatically and can also receive custom metrics from your applications.

Key Characteristics:

  • • Time-ordered set of data points
  • • Published to CloudWatch with timestamp
  • • Organized by namespace and dimensions
  • • Retained for different periods based on age

Namespaces

Namespaces are containers for CloudWatch metrics. Metrics in different namespaces are isolated from each other.

Common AWS Namespaces:

• AWS/EC2
• AWS/RDS
• AWS/S3
• AWS/Lambda
• AWS/ELB
• AWS/DynamoDB

Dimensions

Dimensions are name/value pairs that uniquely identify a metric. You can assign up to 10 dimensions to a metric.

Example Dimensions:

EC2: InstanceId, InstanceType, ImageId
RDS: DBInstanceIdentifier, DatabaseClass
S3: BucketName, StorageType

Metric Types

Default Metrics

Automatically provided by AWS services at no additional charge.

  • • EC2: CPU, Network, Disk (5-minute intervals)
  • • RDS: CPU, Connections, IOPS
  • • S3: Bucket size, Number of objects
  • • Lambda: Invocations, Duration, Errors

Custom Metrics

Application-specific metrics you publish to CloudWatch.

  • • Business metrics (orders, revenue)
  • • Application performance metrics
  • • Custom system metrics
  • • High-resolution metrics (1-second intervals)

Data Retention Periods

Data Point Period Retention Period Description
Less than 60 seconds 3 hours High-resolution custom metrics
60 seconds (1 minute) 15 days Standard resolution metrics
300 seconds (5 minutes) 63 days Default EC2 metrics
3600 seconds (1 hour) 455 days Aggregated hourly data

CloudWatch Statistics

Statistics are metric data aggregations over specified periods of time. CloudWatch provides several statistical functions:

Minimum

Lowest value observed during the specified period

Maximum

Highest value observed during the specified period

Sum

All values submitted for the matching metric added together

Average

Sum / SampleCount

SampleCount

Number of data points used for the statistical calculation

Percentiles

p99, p95, p90, etc. - Value below which a percentage of observations fall

Best Practices

Metric Naming

Use descriptive, consistent naming conventions for custom metrics. Include units in metric names when appropriate.

Dimension Strategy

Use dimensions wisely - they create unique metric streams. Too many dimensions can lead to high cardinality and increased costs.

Cost Optimization

Monitor custom metric usage and API calls. Use appropriate retention periods and consider metric aggregation for cost efficiency.

Monitoring Strategy

Implement a comprehensive monitoring strategy covering infrastructure, application, and business metrics.

Key Takeaways

  • CloudWatch is the central monitoring service for AWS resources and applications
  • Metrics are organized by namespaces and identified by dimensions
  • Data retention varies based on metric resolution and age
  • Statistics provide aggregated views of metric data over time periods
  • Proper dimension strategy is crucial for effective monitoring and cost control
