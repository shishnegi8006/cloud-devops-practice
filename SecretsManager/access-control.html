Access Control & IAM

Permissions and security policies

IAM Policies

Control access using IAM identity-based policies.

  • GetSecretValue - Read secret values
  • CreateSecret - Create new secrets
  • UpdateSecret - Modify existing secrets
  • DeleteSecret - Remove secrets
  • RotateSecret - Trigger rotation

Resource Policies

Attach policies directly to secrets for fine-grained control.

  • Cross-account access
  • Condition-based access
  • Time-based restrictions
  • IP address filtering
  • MFA requirements
