Compliance and monitoring

CloudTrail Logging Deep Dive

All Secrets Manager API calls are automatically logged to AWS CloudTrail, providing comprehensive audit trails for compliance and security monitoring.

Logged Events:

  • GetSecretValue: Secret access attempts (value not logged)
  • CreateSecret: New secret creation
  • UpdateSecret: Secret configuration changes
  • DeleteSecret: Secret deletion requests
  • RotateSecret: Rotation initiation
  • PutSecretValue: Secret value updates

Event Details Include:

  • User identity and source IP
  • Timestamp and region
  • Request parameters (excluding secret values)
  • Response elements
  • Error codes and messages

Security Considerations:

  • Secret values are never logged
  • CloudTrail logs are encrypted
  • Log file integrity validation
  • Cross-region log delivery

CloudWatch Metrics & Alarms

Available Metrics:

SuccessfulRequestLatency

Time taken for successful API requests

ProvisionedThroughputExceeded

Number of requests that exceeded throughput limits

UserErrors

Client-side errors (4xx HTTP status codes)

SystemErrors

Server-side errors (5xx HTTP status codes)

Custom Alarms:

  • High error rates
  • Unusual access patterns
  • Failed rotation attempts
  • Throttling events

Compliance & Governance Framework

Compliance Standards

  • SOC 1, 2, 3: Service Organization Control reports
  • PCI DSS: Payment Card Industry compliance
  • HIPAA: Healthcare data protection
  • ISO 27001: Information security management
  • FedRAMP: Federal risk management program
  • GDPR: General Data Protection Regulation

Monitoring Best Practices

  • Set up CloudWatch dashboards
  • Configure SNS notifications
  • Monitor cross-account access
  • Track rotation success rates
  • Alert on policy changes
  • Review access patterns regularly

Security Monitoring

  • Failed authentication attempts
  • Unusual geographic access
  • High-frequency secret retrieval
  • Policy modification events
  • Cross-region replication changes

Automated Response

  • Lambda-based incident response
  • Automatic secret rotation triggers
  • Security group modifications
  • IAM policy adjustments
  • Notification escalation workflows
