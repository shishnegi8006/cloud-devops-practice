Automatic Rotation Overview

Secrets Manager can automatically rotate secrets using Lambda functions, ensuring credentials are regularly updated without manual intervention.

Rotation Benefits:

  • Security: Regular credential updates reduce breach risk
  • Compliance: Meet regulatory rotation requirements
  • Zero Downtime: Seamless rotation without service interruption
  • Automated: No manual intervention required

Rotation Schedules:

  • 30, 60, 90 days (common intervals)
  • Custom intervals (1-365 days)
  • Immediate rotation on-demand
  • Rotation windows for maintenance

Detailed Rotation Process

Create Version

Generate new secret version with AWSPENDING label

Set Secret

Update service with new credentials

Test Secret

Verify new credentials work correctly

Finish Rotation

Move AWSCURRENT label to new version

Rotation Strategies:

Single User Strategy

  • Updates password for existing user
  • Simpler implementation
  • Brief connection interruption possible
  • Suitable for development environments

Multi User Strategy

  • Creates clone user with same permissions
  • Zero downtime rotation
  • More complex setup required
  • Recommended for production

Error Handling & Monitoring:

  • CloudWatch Logs: Detailed rotation execution logs
  • CloudWatch Metrics: Rotation success/failure metrics
  • SNS Notifications: Alert on rotation failures
  • Rollback: Automatic rollback on test failures
  • Retry Logic: Configurable retry attempts
