Secrets Fundamentals

Core concepts and security best practices

What is AWS Secrets Manager?

AWS Secrets Manager helps you protect secrets needed to access your applications, services, and IT resources. The service enables you to easily rotate, manage, and retrieve database credentials, API keys, and other secrets throughout their lifecycle.

Key Benefits:

  • Centralized Management: Store all secrets in one secure location
  • Automatic Rotation: Built-in rotation for RDS, DocumentDB, Redshift
  • Fine-grained Access: IAM and resource-based policies
  • Encryption: All secrets encrypted at rest and in transit
  • Audit Trail: Complete logging with CloudTrail
  • Cross-Region: Replicate secrets across regions

Use Cases:

  • Database credential management
  • API key storage and rotation
  • Application configuration secrets
  • Third-party service authentication

Types of Secrets

Database Credentials

Username/password pairs for databases

  • RDS (MySQL, PostgreSQL, Oracle, SQL Server)
  • Amazon Aurora
  • Amazon DocumentDB
  • Amazon Redshift

API Keys & Tokens

Authentication credentials for services

  • REST API authentication keys
  • OAuth access tokens
  • JWT tokens
  • Third-party service credentials

Application Secrets

Configuration and runtime secrets

  • Encryption keys
  • License keys
  • Configuration strings
  • Connection strings

Security Architecture

Encryption:

  • At Rest: AES-256 encryption using AWS KMS
  • In Transit: TLS 1.2 for all API calls
  • Key Management: AWS managed or customer managed KMS keys

Access Control:

  • IAM Policies: Identity-based permissions
  • Resource Policies: Secret-level access control
  • VPC Endpoints: Private network access
  • Condition Keys: Time, IP, MFA-based access

Compliance:

  • SOC 1, 2, 3 compliance
  • PCI DSS Level 1
  • HIPAA eligible
  • ISO 27001 certified

Best Practices & Implementation

Security Best Practices:

  • Least Privilege: Grant minimum required permissions
  • Rotation: Enable automatic rotation when possible
  • Monitoring: Set up CloudWatch alarms for access patterns
  • VPC Endpoints: Use for enhanced network security

Naming Conventions:

  • Use hierarchical naming: /app/environment/service
  • Include environment indicators: prod, dev, test
  • Use descriptive names for easy identification

Cost Optimization:

  • Implement caching to reduce API calls
  • Use appropriate rotation intervals
  • Monitor usage with CloudWatch metrics
  • Clean up unused secrets regularly
