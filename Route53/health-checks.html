Health Checks

Monitoring & Failover Mechanisms

Health Check Overview

Route 53 health checks monitor the health and performance of your resources and enable automatic failover.

  • Automatic failover to healthy resources
  • Global health checking from multiple locations
  • Integration with CloudWatch alarms
  • SNS notifications for health status changes

Health Check Types

Endpoint Health Checks

Monitor specific endpoints (HTTP, HTTPS, TCP)

Calculated Health Checks

Combine results using AND, OR, NOT operations

CloudWatch Alarm

Monitor CloudWatch alarms for health status

Configuration

Basic Settings

  • Protocol: HTTP, HTTPS, TCP
  • IP/Domain: Target endpoint
  • Port: Target port number
  • Path: Resource path

Advanced Settings

  • Interval: 30s (standard) or 10s (fast)
  • Failure Threshold: 1-10 consecutive failures
  • String Matching: Response body validation
  • Latency Graphs: Performance monitoring

Global Locations

Route 53 performs health checks from multiple global locations to ensure accurate monitoring.

  • 15+ health checking locations
  • Distributed across continents
  • Majority of locations must pass
  • Default: 18% of checkers must pass
  • Customizable threshold

Exam Strategy Tip

Remember: Health checks are required for failover routing. Use CloudWatch alarms for private resources. Calculated health checks combine multiple checks. Health checks are performed from multiple global locations.
