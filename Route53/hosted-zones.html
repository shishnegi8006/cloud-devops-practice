Hosted Zones

Public & Private DNS Management

What are Hosted Zones?

A hosted zone is a container for DNS records for a particular domain. Route 53 automatically creates NS and SOA records.

Key Components:
  • NS Records: Name servers that respond to DNS queries
  • SOA Record: Start of Authority with zone information
  • DNS Records: A, AAAA, CNAME, MX, TXT, etc.
  • Hosted Zone ID: Unique identifier for the zone

Public Hosted Zones

Public hosted zones respond to DNS queries from the internet for public domains.

Use Cases

  • Public websites
  • Email servers
  • API endpoints
  • CDN distributions

Features

  • Internet-accessible
  • Global DNS resolution
  • Health checks
  • DNSSEC support

Private Hosted Zones

Private hosted zones respond to DNS queries from within specified VPCs for internal domains.

Use Cases

  • Internal applications
  • Database endpoints
  • Microservices
  • Development environments

Requirements

  • Associated with VPC(s)
  • DNS resolution enabled
  • DNS hostnames enabled
  • Cross-account sharing possible

Split-View DNS

Use the same domain name for both public and private hosted zones with different records.

Example Scenario:
Public Zone
api.company.com → 203.0.113.10
(External load balancer)
Private Zone
api.company.com → 10.0.1.100
(Internal server)

Exam Strategy Tip

Remember: Public zones are for internet-facing domains, Private zones require VPC association. Split-view DNS allows same domain for public/private. Each hosted zone has unique NS records.
