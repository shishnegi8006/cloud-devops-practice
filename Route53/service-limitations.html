Route 53 Service Limitations

DNS Service Quotas and Constraints

Hosted Zone Limits

  • Hosted zones per AWS account: 500
  • Records per hosted zone: 10,000
  • VPCs per private hosted zone: 300
  • Private hosted zones per VPC: 1,000
  • Reusable delegation sets per account: 100
  • Name servers per reusable delegation set: 4
  • Hosted zones per reusable delegation set: 100
  • Maximum record name length: 255 octets
  • Maximum record value length: 4,000 characters
  • Maximum records per resource record set: 400

DNS Query Limits

  • Queries per second per hosted zone: No limit
  • Resolver queries per second per IP: 10,000
  • Resolver rules per Region: 1,000
  • Resolver rule associations per VPC: 1,000
  • Resolver endpoints per Region: 4
  • IP addresses per resolver endpoint: 10
  • DNS Firewall rule groups per Region: 1,000
  • DNS Firewall rules per rule group: 1,000
  • DNS Firewall rule group associations per VPC: 5

Health Check Limits

  • Health checks per AWS account: 200
  • Health checkers per health check: 15 (default)
  • Health check regions: 15 global locations
  • Health check interval: 30 seconds (standard), 10 seconds (fast)
  • Health check timeout: 4 seconds (HTTP/HTTPS), 10 seconds (TCP)
  • Failure threshold: 3 consecutive failures
  • String matching length: 255 characters
  • CloudWatch alarms per calculated health check: 255
  • Child health checks per calculated health check: 255

Domain Registration Limits

  • Domains per AWS account: 50 (default), up to 500
  • Domain registration period: 1-10 years (varies by TLD)
  • Domain transfer lock period: 60 days after registration/transfer
  • Name servers per domain: 2-12
  • Domain contacts per domain: 3 (registrant, admin, tech)
  • Domain auto-renewal: Enabled by default
  • Domain privacy protection: Available for most TLDs
  • Domain transfer authorization code validity: 15 days
  • Bulk domain operations: 20 domains per request

Traffic Policy Limits

  • Traffic policies per AWS account: 50
  • Traffic policy versions per policy: 1,000
  • Policy records per AWS account: 5
  • Endpoints per traffic policy: 50
  • Geolocation locations per policy: 100
  • Traffic policy document size: 50 KB
  • Traffic Flow visual editor elements: 50
  • Nested traffic policies depth: 5 levels
  • Traffic policy evaluation time: 100ms timeout

Performance & TTL Limits

  • Minimum TTL: 0 seconds
  • Maximum TTL: 2,147,483,647 seconds (~68 years)
  • Default TTL: 300 seconds (5 minutes)
  • Alias record TTL: Automatically set by target
  • DNS propagation time: Up to 48 hours globally
  • Route 53 response time: <100ms (typical)
  • Anycast network locations: 200+ edge locations
  • DNS over HTTPS (DoH) support: Yes
  • DNSSEC validation: Supported for resolver

Exam Tips for Route 53 Service Limitations

  • Most Route 53 limits are soft limits that can be increased via AWS Support
  • Private hosted zones can be associated with up to 300 VPCs across regions
  • Health checks use 15 global locations by default for redundancy
  • Traffic policies are limited to 50 endpoints but support complex routing logic
  • Domain registration limits start at 50 but can be increased to 500
  • Resolver endpoints are limited to 4 per region with 10 IP addresses each
  • DNS Firewall provides protection with up to 1,000 rules per rule group
