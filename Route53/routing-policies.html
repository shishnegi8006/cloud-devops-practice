Routing Policies

Traffic Management & Distribution

Simple Routing

Routes traffic to a single resource. No health checks or traffic distribution logic.

  • One record per name
  • Multiple values possible
  • Random selection by client
  • No health checks

Weighted Routing

Distributes traffic across multiple resources based on assigned weights.

  • Weight 0-255
  • Percentage calculation
  • A/B testing
  • Blue/green deployments

Latency-Based

Routes traffic to the resource with the lowest latency from user's location.

  • Global applications
  • Performance optimization
  • Multi-region deployments
  • Specify AWS region

Geolocation

Routes traffic based on the geographic location of the user making the request.

  • Content localization
  • Compliance requirements
  • Regional restrictions
  • Continent/Country/State

Failover

Routes traffic to primary resource, with automatic failover to secondary if primary fails.

  • Disaster recovery
  • High availability
  • Health checks mandatory
  • Primary and secondary records

Multivalue Answer

Returns multiple healthy IP addresses in response to DNS queries.

  • Up to 8 healthy records
  • Health checks supported
  • Client-side load balancing
  • Simple failover

Exam Strategy Tip

Remember: Simple = single resource, Weighted = traffic distribution, Latency = best performance, Geolocation = user location, Failover = primary/secondary with health checks.
