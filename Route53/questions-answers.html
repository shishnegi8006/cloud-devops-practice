Route 53 Practice Questions

Question 1

Which Route 53 routing policy should you use to distribute traffic across multiple resources based on assigned weights?

A) Simple routing
B) Weighted routing
C) Latency-based routing
D) Geolocation routing
Correct Answer: B) Weighted routing

Explanation: Weighted routing allows you to distribute traffic across multiple resources based on assigned weights (0-255). This is perfect for A/B testing, blue/green deployments, and gradual traffic migration. Simple routing doesn't support traffic distribution, latency-based routing routes based on performance, and geolocation routing routes based on user location.
Question 2

What is the main difference between public and private hosted zones in Route 53?

A) Public zones are free, private zones are paid
B) Public zones respond to internet queries, private zones respond to VPC queries
C) Public zones support health checks, private zones don't
D) Public zones have higher TTL limits
Correct Answer: B) Public zones respond to internet queries, private zones respond to VPC queries

Explanation: Public hosted zones respond to DNS queries from the internet for public domains, while private hosted zones respond to DNS queries from within specified VPCs for internal domains. Both types have the same pricing structure, both can use health checks, and TTL limits are the same for both.
Question 3

Which health check type should you use to monitor private resources that are not accessible from the internet?

A) Endpoint health checks
B) CloudWatch alarm health checks
C) Calculated health checks only
D) HTTP health checks with VPC endpoints
Correct Answer: B) CloudWatch alarm health checks

Explanation: CloudWatch alarm health checks are the best way to monitor private resources that are not accessible from the internet. You can create CloudWatch alarms based on custom metrics from your private resources, and Route 53 can monitor these alarms to determine health status. Endpoint health checks cannot reach private resources, and calculated health checks would still need a way to monitor the private resources.
Question 4

What is the purpose of TTL (Time To Live) in DNS records?

A) To determine the priority of DNS records
B) To specify how long DNS records are cached by resolvers
C) To set the expiration date of the domain
D) To configure health check intervals
Correct Answer: B) To specify how long DNS records are cached by resolvers

Explanation: TTL (Time To Live) determines how long DNS records are cached by resolvers and browsers before they query the authoritative nameserver again. Lower TTL values mean more frequent queries but faster propagation of changes. Higher TTL values reduce query load but slow down change propagation. TTL is not related to record priority, domain expiration, or health check intervals.
Question 5

Which Route 53 feature provides protection against DNS-based threats by filtering malicious domains?

A) Route 53 Resolver
B) Route 53 Resolver DNS Firewall
C) DNSSEC
D) Traffic Flow
Correct Answer: B) Route 53 Resolver DNS Firewall

Explanation: Route 53 Resolver DNS Firewall provides protection against DNS-based threats by filtering and monitoring DNS queries. It can block access to known malicious domains, prevent data exfiltration via DNS, and enforce corporate internet usage policies. Route 53 Resolver provides DNS resolution, DNSSEC provides authentication, and Traffic Flow is for complex routing policies.
Question 6

A company wants to route traffic to the closest AWS region based on network latency. Which routing policy should they use?

A) Geolocation routing
B) Latency-based routing
C) Geoproximity routing
D) Weighted routing
Correct Answer: B) Latency-based routing

Explanation: Latency-based routing routes traffic to the AWS region that provides the lowest latency for the user. Route 53 measures latency between users and AWS regions and automatically routes to the best-performing region. Geolocation routing is based on user's geographic location, geoproximity uses geographic location with bias, and weighted routing distributes traffic by percentage.
Question 7

What happens when you enable DNSSEC for a Route 53 hosted zone?

A) DNS queries are encrypted in transit
B) DNS responses are digitally signed for authentication
C) DNS queries are blocked from malicious sources
D) DNS resolution speed is increased
Correct Answer: B) DNS responses are digitally signed for authentication

Explanation: DNSSEC (DNS Security Extensions) provides authentication and integrity verification for DNS responses by digitally signing them. This protects against DNS spoofing and cache poisoning attacks. DNSSEC does not encrypt DNS queries (that would be DNS over HTTPS/TLS), does not block malicious sources (that's DNS Firewall), and does not increase resolution speed.
Question 8

Which record type cannot be used for the root domain (apex domain)?

A) A record
B) CNAME record
C) MX record
D) TXT record
Correct Answer: B) CNAME record

Explanation: CNAME records cannot be used for the root domain (apex domain) because DNS standards prohibit CNAME records from coexisting with other record types at the same name, and the root domain must have SOA and NS records. For root domains, you must use A records (for IPv4) or AAAA records (for IPv6). Route 53 provides alias records as an alternative that works like CNAME but can be used at the root domain.
Question 9

What is the main purpose of Route 53 Resolver endpoints?

A) Enable hybrid DNS resolution between AWS and on-premises networks
B) Improve DNS query performance
C) Provide DNS caching
D) Enable DNSSEC validation
Correct Answer: A) Enable hybrid DNS resolution between AWS and on-premises networks

Explanation: Route 53 Resolver endpoints enable hybrid DNS resolution between AWS and on-premises networks. Inbound endpoints allow on-premises DNS resolvers to forward queries to Route 53 Resolver, while outbound endpoints allow VPCs to forward queries to on-premises DNS resolvers. This enables seamless DNS resolution across hybrid cloud environments.
Question 10

A company needs to implement disaster recovery with automatic failover. Which routing policy requires health checks?

A) Simple routing
B) Weighted routing
C) Failover routing
D) Geolocation routing
Correct Answer: C) Failover routing

Explanation: Failover routing is specifically designed for disaster recovery scenarios and requires health checks to be configured. It routes traffic to a primary resource when healthy, and automatically fails over to a secondary resource when the primary fails health checks. Simple routing doesn't support health checks, while weighted and geolocation routing support health checks but don't require them.
