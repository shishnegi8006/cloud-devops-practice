Elastic Load Balancer Types

Application Load Balancer (ALB)

Layer 7 load balancer for HTTP/HTTPS traffic with advanced routing.

Use Case:

Web applications, microservices, container-based apps

Features: Path-based routing, host-based routing, WebSocket support

Network Load Balancer (NLB)

Layer 4 load balancer for TCP/UDP traffic with ultra-high performance.

Use Case:

Gaming applications, IoT, real-time communications

Features: Static IP, extreme performance, preserve source IP

Gateway Load Balancer (GWLB)

Layer 3 load balancer for deploying third-party network appliances.

Use Case:

Firewalls, intrusion detection, deep packet inspection

Features: GENEVE protocol, transparent network gateway

Auto Scaling Group Features

Dynamic Scaling

Automatically adjust capacity based on demand using CloudWatch metrics.

Use Case:

Handle traffic spikes during peak hours or events

Types: Target Tracking, Step Scaling, Simple Scaling

Scheduled Scaling

Scale based on predictable patterns using scheduled actions.

Use Case:

Scale up before business hours, scale down at night

Benefit: Proactive scaling for known patterns

Health Checks

Monitor instance health and replace unhealthy instances automatically.

Use Case:

Maintain application availability by replacing failed instances

Types: EC2 health checks, ELB health checks

Launch Templates

Define instance configuration including AMI, instance type, and security groups.

Use Case:

Standardize instance configuration across environments

Features: Versioning, mixed instance types, Spot instances

Load Balancer Integration

Seamlessly integrate with ELB for automatic traffic distribution.

Use Case:

Distribute traffic across scaled instances automatically

Benefit: Automatic registration/deregistration of instances

Instance Protection

Protect specific instances from scale-in activities.

Use Case:

Protect instances running critical batch jobs

Types: Scale-in protection, instance termination protection

Best Practices

Load Balancer Best Practices

  • Use ALB for HTTP/HTTPS applications
  • Enable access logs for troubleshooting
  • Configure appropriate health check settings
  • Use SSL/TLS termination at load balancer

Auto Scaling Best Practices

  • Use target tracking scaling policies
  • Set appropriate cooldown periods
  • Monitor scaling activities and metrics
  • Use mixed instance types for cost optimization
