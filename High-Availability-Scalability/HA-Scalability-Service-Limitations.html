HA & Scalability Service Limitations

ELB, ASG & High Availability Quotas and Constraints

ELB Limits

  • Application Load Balancers per Region: 50
  • Network Load Balancers per Region: 50
  • Gateway Load Balancers per Region: 50
  • Classic Load Balancers per Region: 20
  • Targets per ALB: 1,000
  • Targets per NLB: 3,000
  • Listeners per ALB: 50
  • Listeners per NLB: 50
  • Rules per ALB listener: 100
  • Target groups per Region: 3,000

Auto Scaling Group Limits

  • Auto Scaling groups per Region: 500
  • Launch configurations per Region: 200
  • Launch templates per Region: 5,000
  • Launch template versions per template: 10,000
  • Scaling policies per ASG: 50
  • Scheduled actions per ASG: 125
  • Lifecycle hooks per ASG: 50
  • SNS topics per ASG: 10
  • Load balancers per ASG: 50
  • Target groups per ASG: 50

Performance & Capacity Limits

  • ALB requests per second: 25,000 (can burst higher)
  • NLB connections per second: Millions
  • ALB concurrent connections: 55,000
  • NLB concurrent connections: Millions
  • ASG scaling activities per minute: 5
  • ASG cooldown period: 0-3600 seconds
  • Target health check interval: 5-300 seconds
  • Unhealthy threshold: 2-10 consecutive checks
  • Healthy threshold: 2-10 consecutive checks

SSL/TLS & Security Limits

  • SSL certificates per ALB: 25
  • SSL certificates per NLB: 25
  • SSL certificates per CLB: 1
  • SSL negotiation per second (ALB): 25
  • SSL negotiation per second (NLB): 50,000
  • Security groups per ELB: 5
  • WAF web ACLs per ALB: 1
  • Client IP preservation: NLB only (default)
  • Cross-zone load balancing: ALB (always), NLB (optional)

Availability Zone Limits

  • Availability Zones per Region: 2-6 (varies by region)
  • Minimum AZs for ALB: 2
  • Minimum AZs for NLB: 1
  • Maximum AZs per ELB: All available in region
  • Subnets per AZ for ELB: 1
  • ASG minimum AZs: 1
  • ASG maximum AZs: All available in region
  • Cross-AZ failover time: 1-2 minutes (typical)
  • Health check grace period: 0-7200 seconds

Monitoring & Logging Limits

  • CloudWatch metrics retention: 15 months
  • Access logs retention: Configurable in S3
  • Connection logs (NLB): Configurable in S3
  • CloudWatch alarms per ASG: No specific limit
  • Custom metrics per ASG: No specific limit
  • ELB access log frequency: Every 5 or 60 minutes
  • Request tracing (ALB): X-Amzn-Trace-Id header
  • CloudTrail API logging: All ELB and ASG APIs

Exam Tips for HA & Scalability Service Limitations

  • Most ELB and ASG limits are soft limits that can be increased via AWS Support
  • ALB requires minimum 2 AZs, NLB can operate with 1 AZ but 2+ recommended
  • NLB provides higher performance with millions of connections vs ALB's 55,000
  • ASG scaling activities are limited to 5 per minute to prevent rapid scaling
  • Cross-zone load balancing is always enabled for ALB, optional for NLB
  • Target group limits affect how you design multi-tier applications
  • SSL certificate limits may require certificate consolidation or SNI
