ELB Fundamentals

Master Elastic Load Balancer types, configurations, and routing strategies

Load Balancer Overview

What is Load Balancing?

Distribution: Spreads incoming requests across multiple targets

High Availability: Eliminates single points of failure

Performance: Improves response times and throughput

AWS ELB Benefits

Managed Service: AWS handles infrastructure and scaling

Health Checks: Automatic unhealthy target detection

Security: SSL/TLS termination and WAF integration

Application Load Balancer (ALB)

Layer 7 Load Balancing

Operates at the application layer with HTTP/HTTPS protocol awareness.

Features:

  • • Content-based routing
  • • Host-based routing
  • • Path-based routing
  • • HTTP header routing
Best For: Web applications, microservices

Advanced Routing

Sophisticated routing rules based on request attributes.

Routing Types:

  • • Path patterns (/api/*, /images/*)
  • • Host headers (api.example.com)
  • • HTTP methods (GET, POST)
  • • Query strings (?version=v2)
Use Case: Microservices architecture

Target Groups

Logical grouping of targets for health checks and routing.

Target Types:

  • • EC2 instances
  • • IP addresses
  • • Lambda functions
  • • Application Load Balancers
Health Check: HTTP/HTTPS endpoint monitoring

SSL/TLS Termination

Handles SSL/TLS encryption and decryption at the load balancer.

Features:

  • • ACM certificate integration
  • • SNI support
  • • Security policy configuration
  • • HTTP to HTTPS redirect
Benefit: Offloads SSL processing from targets

Sticky Sessions

Route requests from same client to same target consistently.

Methods:

  • • Duration-based cookies
  • • Application-controlled cookies
  • • Configurable duration
Use Case: Session-dependent applications

WAF Integration

Web Application Firewall protection against common attacks.

Protection:

  • • SQL injection
  • • Cross-site scripting (XSS)
  • • DDoS mitigation
  • • Rate limiting
Integration: Seamless WAF rule application

Network Load Balancer (NLB)

Layer 4 Load Balancing

Operates at the transport layer with TCP/UDP protocol support.

Protocols:

  • • TCP (Transmission Control Protocol)
  • • UDP (User Datagram Protocol)
  • • TLS (Transport Layer Security)
Best For: High-performance applications

Ultra-High Performance

Handles millions of requests per second with ultra-low latency.

Performance:

  • • Millions of RPS capability
  • • Sub-millisecond latency
  • • Static IP addresses
  • • Elastic IP support
Use Case: Gaming, IoT, real-time applications

Connection Preservation

Preserves client IP addresses and connection details.

Features:

  • • Source IP preservation
  • • Flow hash algorithm
  • • Connection multiplexing
Benefit: Maintains client context

Health Checks

TCP and HTTP/HTTPS health check options for target monitoring.

Types:

  • • TCP connection checks
  • • HTTP/HTTPS endpoint checks
  • • Configurable intervals
Monitoring: CloudWatch integration

Static IP & Elastic IP

Provides static IP addresses and supports Elastic IP assignment.

IP Features:

  • • One static IP per AZ
  • • Elastic IP assignment
  • • Whitelisting support
Use Case: Firewall rules, DNS records

TLS Termination

Supports TLS termination for encrypted traffic handling.

TLS Features:

  • • TLS 1.0, 1.1, 1.2, 1.3 support
  • • Certificate management
  • • SNI support
Benefit: Offloads TLS processing

Gateway Load Balancer (GWLB)

Layer 3 Load Balancing

Operates at the network layer for transparent traffic inspection.

Purpose:

  • • Network appliance load balancing
  • • Traffic inspection and filtering
  • • Security appliance scaling
Best For: Security appliances, firewalls

Transparent Proxy

Provides transparent traffic forwarding without modifying packets.

Features:

  • • GENEVE protocol support
  • • Packet preservation
  • • Bidirectional traffic flow
Benefit: Invisible to applications

Third-Party Integration

Integrates with third-party network and security appliances.

Appliances:

  • • Firewalls (Palo Alto, Fortinet)
  • • IDS/IPS systems
  • • Deep packet inspection
Use Case: Security service chaining

Load Balancing Algorithms

ALB Algorithms

Round Robin: Requests distributed evenly across targets

Least Outstanding Requests: Routes to target with fewest active requests

NLB Algorithms

Flow Hash: Uses 5-tuple hash for consistent routing

Connection Affinity: Maintains connection to same target

Monitoring & Logging

CloudWatch Metrics

Comprehensive metrics for load balancer performance monitoring.

Key Metrics:

  • • RequestCount
  • • TargetResponseTime
  • • HTTPCode_Target_2XX_Count
  • • UnHealthyHostCount

Access Logs

Detailed request logs stored in S3 for analysis and troubleshooting.

Log Information:

  • • Request/response details
  • • Client IP addresses
  • • Response times
  • • Error codes

Connection Logs

NLB connection flow logs for network-level troubleshooting.

Flow Data:

  • • Connection start/end times
  • • Bytes transferred
  • • Connection state

ELB Best Practices

Performance Optimization

Pre-warming: Contact AWS for expected high traffic

Health checks: Configure appropriate intervals and thresholds

Connection draining: Enable for graceful instance replacement

Security Best Practices

Security groups: Restrict access to necessary ports only

SSL/TLS: Use latest security policies and certificates

Monitoring: Enable access logs and CloudWatch alarms
