RDS Multi-AZ Deployment

Question: A company needs to deploy a MySQL database that can automatically failover to a standby instance in case of primary instance failure. The solution should provide zero data loss and minimal downtime. Which AWS service configuration meets these requirements?

A) RDS MySQL with read replicas in multiple AZs
B) RDS MySQL with Multi-AZ deployment
C) Aurora MySQL with Aurora Serverless
D) EC2 instances with MySQL and EBS snapshots

Correct Answer: B) RDS MySQL with Multi-AZ deployment

Explanation:

  • • Multi-AZ deployment provides synchronous replication to a standby instance in a different AZ
  • • Automatic failover occurs within 1-2 minutes with zero data loss
  • • Option A is incorrect: Read replicas use asynchronous replication and don't provide automatic failover
  • • Option C is incorrect: Aurora Serverless doesn't specifically address failover requirements
  • • Option D is incorrect: Requires manual intervention and doesn't meet zero data loss requirement

ElastiCache Engine Selection

Question: An application requires a caching solution that supports complex data structures, pub/sub messaging, and data persistence. The cache should also support clustering for horizontal scaling. Which ElastiCache engine should be used?

A) Memcached with Auto Discovery
B) Redis with Cluster Mode enabled
C) Memcached with multiple nodes
D) Redis with Cluster Mode disabled

Correct Answer: B) Redis with Cluster Mode enabled

Explanation:

  • • Redis supports complex data structures (lists, sets, hashes), pub/sub, and data persistence
  • • Cluster Mode enabled provides horizontal scaling across multiple shards
  • • Option A & C are incorrect: Memcached doesn't support complex data structures or persistence
  • • Option D is incorrect: Cluster Mode disabled limits scaling to vertical scaling only

Aurora Global Database

Question: A global application needs a database solution that can serve read traffic from multiple regions with low latency while maintaining a single write region. The solution should provide disaster recovery with RTO < 1 minute. What is the best approach?

A) RDS with cross-region read replicas
B) Aurora Global Database
C) DynamoDB Global Tables
D) Multiple RDS instances with DMS replication

Correct Answer: B) Aurora Global Database

Explanation:

  • • Aurora Global Database provides cross-region replication with < 1 second lag
  • • Fast failover with RTO < 1 minute meets the disaster recovery requirement
  • • Single write region with multiple read regions matches the requirement
  • • Option A is incorrect: RDS cross-region replicas have higher lag and slower failover
  • • Option C is incorrect: DynamoDB Global Tables support multi-write, not single write
  • • Option D is incorrect: Complex setup with higher operational overhead

Database Migration Strategy

Question: A company wants to migrate an Oracle database to Aurora PostgreSQL with minimal downtime. The migration involves schema conversion and ongoing data synchronization. Which combination of AWS services should be used?

A) AWS DataSync + AWS Glue
B) AWS Schema Conversion Tool (SCT) + AWS Database Migration Service (DMS)
C) AWS DMS only
D) Native Oracle export/import tools

Correct Answer: B) AWS Schema Conversion Tool (SCT) + AWS Database Migration Service (DMS)

Explanation:

  • • Heterogeneous migration (Oracle to PostgreSQL) requires both SCT and DMS
  • • SCT converts the database schema and identifies conversion issues
  • • DMS performs the data migration with Full Load + CDC for minimal downtime
  • • Option A is incorrect: DataSync is for file transfer, not database migration
  • • Option C is incorrect: DMS alone cannot handle schema conversion
  • • Option D is incorrect: Native tools don't support cross-engine migration

Database Security & Encryption

Question: A financial services company requires encryption at rest for their RDS database and wants to use their own encryption keys. They also need to rotate keys automatically. What is the most appropriate solution?

A) Use AWS managed keys with automatic rotation
B) Use customer managed KMS keys with automatic rotation enabled
C) Use CloudHSM for key management
D) Implement application-level encryption

Correct Answer: B) Use customer managed KMS keys with automatic rotation enabled

Explanation:

  • • Customer managed KMS keys allow the company to control their own encryption keys
  • • Automatic rotation can be enabled for customer managed keys (annual rotation)
  • • RDS encryption at rest integrates seamlessly with KMS
  • • Option A is incorrect: AWS managed keys don't provide customer control
  • • Option C is incorrect: CloudHSM is more complex and not required for this use case
  • • Option D is incorrect: Application-level encryption adds complexity and doesn't meet the requirement
