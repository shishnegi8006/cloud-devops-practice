ElastiCache Deep Dive

In-Memory Caching & Performance Optimization

What is Amazon ElastiCache?

Amazon ElastiCache is a fully managed in-memory caching service that supports Redis, Valkey, and Memcached engines. It improves application performance by storing frequently accessed data in memory, reducing database load and latency.

Key Benefits: Sub-millisecond latency, automatic failover, backup and restore, monitoring and metrics, security features.

Redis

Advanced Features

  • • Data persistence (RDB snapshots, AOF)
  • • Complex data structures (lists, sets, hashes)
  • • Pub/Sub messaging
  • • Lua scripting support
  • • Geospatial data support

High Availability

  • • Multi-AZ with automatic failover
  • • Read replicas for scaling reads
  • • Redis Cluster for horizontal scaling
  • • Backup and restore capabilities

Use Cases

  • • Session store
  • • Real-time analytics
  • • Gaming leaderboards
  • • Chat/messaging applications

Valkey

Open Source Alternative

  • • Redis-compatible open source fork
  • • Same API and data structures as Redis
  • • Community-driven development
  • • No licensing restrictions
  • • Enhanced security features

Key Benefits

  • • Drop-in Redis replacement
  • • Improved performance optimizations
  • • Enhanced memory management
  • • Better security defaults

Use Cases

  • • Redis migration scenarios
  • • Cost-sensitive applications
  • • Open source compliance requirements
  • • High-performance caching

Memcached

Simple & Fast

  • • Simple key-value store
  • • Multi-threaded architecture
  • • Extremely low latency
  • • Minimal memory overhead
  • • Easy horizontal scaling

Scaling

  • • Auto Discovery for dynamic scaling
  • • Consistent hashing
  • • Node addition/removal
  • • No data persistence

Use Cases

  • • Database query caching
  • • Web session caching
  • • Object caching
  • • Simple distributed caching

Redis Cluster Architecture

Cluster Mode Disabled (Single Shard)

  • • One primary node with up to 5 read replicas
  • • Asynchronous replication
  • • Automatic failover to replica
  • • Multi-AZ deployment supported
  • • Backup and restore available

Cluster Mode Enabled (Multiple Shards)

  • • Up to 500 nodes (90 shards × 5 replicas + 90 primaries)
  • • Data partitioned across shards
  • • Horizontal scaling capability
  • • Online resharding support
  • • Higher availability and performance

Security Features

Encryption

  • • Encryption at rest with KMS
  • • Encryption in transit (TLS)
  • • Auth tokens for Redis

Network Security

  • • VPC deployment
  • • Security groups
  • • Subnet groups

Access Control

  • • IAM authentication (Redis 6.0+)
  • • Redis AUTH command
  • • Role-based access control

Monitoring & Maintenance

CloudWatch Metrics

  • • CPU utilization
  • • Memory usage
  • • Network I/O
  • • Cache hit ratio

Maintenance

  • • Automatic software patching
  • • Maintenance windows
  • • Engine version upgrades

Notifications

  • • SNS notifications
  • • CloudWatch alarms
  • • Event notifications

Best Practices & Performance Tips

Design Patterns

  • • Lazy loading (cache-aside)
  • • Write-through caching
  • • Write-behind caching
  • • TTL for data expiration

Performance

  • • Choose appropriate node types
  • • Monitor memory usage
  • • Use connection pooling
  • • Optimize data structures

Scaling

  • • Horizontal scaling with sharding
  • • Read replicas for read scaling
  • • Monitor cache hit ratios
  • • Plan for peak traffic

SAA-C03 Exam Key Points

  • • Engine Selection: Redis for advanced features, Valkey for open-source compatibility, Memcached for simple caching
  • • Multi-AZ: Available for Redis and Valkey, provides automatic failover
  • • Backup: Redis and Valkey support backup/restore, Memcached does not
  • • Scaling: Redis Cluster for horizontal scaling, read replicas for read scaling
  • • Security: VPC deployment, encryption at rest/transit, IAM authentication
  • • Use Cases: Session store, real-time analytics, gaming, chat applications
