Database Security Fundamentals

AWS provides multiple layers of security for database services including network isolation, encryption, access controls, and compliance frameworks. Security is a shared responsibility between AWS and customers.

Security Pillars: Identity & Access Management, Network Security, Encryption, Monitoring & Auditing, Compliance.

Access Control & Authentication

IAM Database Authentication

  • • Token-based auth: 15-minute tokens
  • • No passwords: Uses IAM credentials
  • • Supported engines: MySQL, PostgreSQL, Aurora
  • • SSL required: Encrypted connections only
  • • IAM policies: Fine-grained permissions

Database User Management

  • • Master user: Administrative privileges
  • • Database users: Application-specific accounts
  • • Password policies: Complexity requirements
  • • Role-based access: Principle of least privilege
  • • Account rotation: Regular password updates

Secrets Manager Integration

  • • Automatic rotation: Scheduled password changes
  • • Secure storage: Encrypted credential storage
  • • Application integration: SDK and API access
  • • Cross-region: Multi-region secret replication
  • • Audit trail: Access logging and monitoring

SAA-C03 Exam Key Points

  • • Encryption at Rest: Must be enabled at creation time, cannot be added later
  • • IAM DB Auth: Uses tokens, requires SSL, supports MySQL/PostgreSQL/Aurora
  • • VPC Security: Private subnets, security groups, NACLs for defense in depth
  • • Compliance: AWS provides infrastructure compliance, customer handles data
  • • Monitoring: CloudTrail for API calls, Database Activity Streams for SQL
  • • Secrets: Use Secrets Manager for automatic rotation and secure storage
