Database Service Limitations

RDS, Aurora & ElastiCache Quotas and Constraints

RDS Limits

  • DB instances per Region: 40 (default)
  • DB subnet groups per Region: 50
  • Subnets per DB subnet group: 20
  • DB parameter groups per Region: 50
  • DB option groups per Region: 20
  • DB security groups per Region: 25
  • VPC security groups per DB instance: 5
  • Read replicas per source DB: 5 (15 for Aurora)
  • Automated backups retention: 0-35 days
  • Manual snapshots per Region: 100

Aurora Limits

  • Aurora clusters per Region: 40
  • Aurora instances per cluster: 15
  • Aurora Serverless v1 concurrent connections: 1,000
  • Aurora Serverless v2 ACU range: 0.5-128 ACUs
  • Aurora Global Database regions: 5 secondary regions
  • Aurora Global Database clusters per primary: 5
  • Aurora cluster volume size: 128 TiB
  • Aurora MySQL connections: 16,000 (max)
  • Aurora PostgreSQL connections: 8,192 (max)
  • Cross-region automated backups: 35 days retention

ElastiCache Redis Limits

  • Nodes per cluster (cluster mode disabled): 6
  • Nodes per shard (cluster mode enabled): 6
  • Shards per cluster: 500
  • Total nodes per cluster: 3,000
  • Parameter groups per Region: 100
  • Subnet groups per Region: 150
  • Subnets per subnet group: 20
  • Security groups per cache cluster: 50
  • Backup retention period: 35 days (max)
  • Manual snapshots per cluster: 20

ElastiCache Memcached Limits

  • Nodes per cluster: 300
  • Parameter groups per Region: 100
  • Subnet groups per Region: 150
  • Subnets per subnet group: 20
  • Security groups per cache cluster: 50
  • Connection overhead per node: ~100 bytes
  • Maximum item size: 1 MB
  • Maximum key length: 250 bytes
  • Maximum connections per node: 65,000
  • No backup/restore capability

Storage & Performance Limits

  • RDS General Purpose SSD: 20 GiB - 64 TiB
  • RDS Provisioned IOPS SSD: 100 GiB - 64 TiB
  • RDS Magnetic storage: 20 GiB - 3 TiB
  • Aurora storage: 10 GiB - 128 TiB (auto-scaling)
  • RDS max IOPS: 80,000 (io2), 40,000 (io1)
  • Aurora max IOPS: 200,000+ (varies by instance)
  • ElastiCache Redis max memory: 317.77 GiB (r6g.16xlarge)
  • ElastiCache Memcached max memory: 317.77 GiB (r6g.16xlarge)

Network & Security Limits

  • DB instance network performance: Up to 25 Gbps
  • ElastiCache network performance: Up to 25 Gbps
  • RDS Proxy max connections: 200,000
  • RDS Proxy max DB connections: 100% of max_connections
  • SSL/TLS encryption: Supported for all engines
  • Encryption at rest: Available for all services
  • VPC security groups: 5 per DB instance/cluster
  • Cross-AZ data transfer: Charged for RDS, free for Aurora

Exam Tips for Database Service Limitations

  • Most database limits are soft limits that can be increased via AWS Support
  • Aurora has higher read replica limits (15) compared to RDS (5)
  • ElastiCache Redis supports clustering and persistence, Memcached does not
  • Aurora storage auto-scales up to 128 TiB, RDS requires manual scaling
  • RDS Proxy helps manage connection limits for serverless and Lambda workloads
  • Consider Aurora Serverless v2 for variable workloads with ACU scaling
  • ElastiCache cluster mode affects node and shard limitations significantly
